# 第4章 核心数据模型

> data 层定义了整个应用的领域模型。BangumiItem 是最核心的数据结构，
> 围绕它展开的还有 History/Progress（观看历史）、CollectedBangumi（收藏）、
> DownloadRecord/DownloadEpisode（下载）等模型。
> 这些模型是 UI 层与服务层之间的数据契约。

---

## 4.1 Hive 持久化概述

Kazumi 使用 **Hive CE**（轻量级 NoSQL 数据库）进行本地持久化。

### Hive 工作原理

```
1. 在模型类上加注解：
   @HiveType(typeId: 0)     → 注册类型
   @HiveField(0) int id;    → 注册字段

2. 运行 build_runner 生成适配器：
   flutter pub run build_runner build
   → bangumi_item.g.dart（自动生成的序列化/反序列化代码）

3. 在 HiveRegistrar 中注册所有适配器：
   hive_registrar.g.dart → 自动注册 9 个 TypeAdapter

4. 读写数据：
   await box.put(key, bangumiItem);     → 写入
   BangumiItem item = box.get(key);     → 读取
```

### Hive vs SQLite

| 对比项 | Hive | SQLite |
|--------|------|--------|
| 读写速度 | 极快（二进制序列化） | 较快（SQL 解析） |
| 查询能力 | 仅按 key 查询 | 支持复杂 SQL 查询 |
| 数据格式 | 键值对（类似 Redis） | 关系表 |
| 依赖 | 纯 Dart | 需要原生库 |
| 适用场景 | 缓存、配置、简单存储 | 复杂查询、关系数据 |

**Kazumi 用 Hive 存储番剧数据、收藏、历史、下载记录——都是按 key 读写的场景，不需要复杂查询。**

## 4.3 ★ BangumiItem — 最核心的数据模型

> 源码：`lib/modules/bangumi/bangumi_item.dart`

```dart
@HiveType(typeId: 0)
class BangumiItem {
    @HiveField(0)  int id;                    // Bangumi 条目 ID
    @HiveField(1)  int type;                  // 类型：2=动画, 3=音乐, 4=游戏, 6=三次元
    @HiveField(2)  String name;               // 日文名
    @HiveField(3)  String nameCn;             // 中文名（可能为空，回退到日文名）
    @HiveField(4)  String summary;            // 简介/描述
    @HiveField(5)  String airDate;            // 播出日期（YYYY-MM-DD）
    @HiveField(6)  int airWeekday;            // 播出星期几（1=周一, 7=周日）
    @HiveField(7)  int rank;                  // 排名（越小越靠前）
    @HiveField(8)  Map<String, String> images;// 图片 URL（large/common/medium/small/grid）
    @HiveField(9)  List<BangumiTag> tags;     // 标签列表
    @HiveField(10) List<String> alias;        // 别名列表
    @HiveField(11) double ratingScore;        // 评分（0.0 - 10.0）
    @HiveField(12) int votes;                 // 总投票数
    @HiveField(13) List<int> votesCount;      // 各分数投票分布（索引 0=1分, 9=10分）
    @HiveField(14) String info;               // 附加信息

    // 不持久化，每次从 API 获取
    BangumiInterest? interest;                // 用户收藏状态
}
```

### API 版本兼容

Bangumi 有两个 API，JSON 格式不同：

```dart
// api.bgm.tv（旧版）
{ "name_cn": "进击的巨人", "infobox": [{"key": "别名", "value": "AOT"}] }
{ "rating": { "count": {"1": 10, "2": 20} } }
{ "date": "2013-04-07" }

// next.bgm.tv（新版）
{ "nameCN": "进击的巨人", "infobox": [{"key": "别名", "values": [{"v": "AOT"}]}] }
{ "rating": { "count": [10, 20] } }
{ "airtime": { "date": "2013-04-07" } }
```

`fromJson` 通过级联回退处理两种格式：

```dart
// 中文名：name_cn → nameCN → name
nameCn: (json['name_cn'] ?? '') == ''
    ? (((json['nameCN'] ?? '') == '') ? json['name'] : json['nameCN'])
    : json['name_cn'],

// 图片：images 对象 → 单张 image
images: Map<String, String>.from(json['images'] ?? { "large": json['image'] }),
```

## 4.6 History + Progress — 观看历史

> 源码：`lib/modules/history/history_module.dart`

```dart
@HiveType(typeId: 1)
class History {
    @HiveField(0) Map<int, Progress> progresses;  // 集数 → 播放进度
    @HiveField(1) int lastWatchEpisode;           // 最后观看集数
    @HiveField(2) String adapterName;             // 规则名（如 "AGE"）
    @HiveField(3) BangumiItem bangumiItem;        // 番剧信息
    @HiveField(4) DateTime lastWatchTime;         // 最后观看时间
    @HiveField(5) String lastSrc;                 // 最后播放地址
    @HiveField(6) String lastWatchEpisodeName;    // 最后观看集名

    String get key => adapterName + bangumiItem.id.toString();  // 复合键
}

@HiveType(typeId: 2)
class Progress {
    @HiveField(0) int episode;           // 集数
    @HiveField(1) int road;              // 播放源
    @HiveField(2) int _progressInMilli;  // 播放位置（毫秒）

    Duration get progress => Duration(milliseconds: _progressInMilli);
}
```

### 数据关系

```
History (一个番剧的观看记录)
  ├── bangumiItem: BangumiItem      ← 番剧信息
  ├── adapterName: "AGE"            ← 用哪个规则看的
  ├── lastWatchEpisode: 3           ← 看到第几集
  ├── lastWatchTime: 2024-01-15     ← 最后观看时间
  └── progresses: {                 ← 每集的播放进度
        1: Progress(episode=1, road=0, progress=12:30),
        2: Progress(episode=2, road=0, progress=24:15),
        3: Progress(episode=3, road=0, progress=05:00),
      }
```

**key = adapterName + bangumiItem.id**，同一个番剧用不同规则看会有不同的历史记录。

## 4.10 DownloadRecord + DownloadEpisode — 下载记录

> 源码：`lib/modules/download/download_module.dart`

```dart
@HiveType(typeId: 7)
class DownloadRecord {
    @HiveField(0) int bangumiId;                        // Bangumi ID
    @HiveField(1) String bangumiName;                   // 番剧名
    @HiveField(2) String bangumiCover;                  // 封面图
    @HiveField(3) String pluginName;                    // 规则名
    @HiveField(4) Map<int, DownloadEpisode> episodes;   // 集数 → 下载状态
    @HiveField(5) DateTime createdAt;                   // 创建时间

    String get key => '${pluginName}_$bangumiId';
}

@HiveType(typeId: 8)
class DownloadEpisode {
    @HiveField(0)  int episodeNumber;       // 集数
    @HiveField(1)  String episodeName;      // 集名
    @HiveField(2)  int road;                // 播放源
    @HiveField(3)  int status;              // 下载状态（DownloadStatus）
    @HiveField(4)  double progressPercent;  // 进度百分比
    @HiveField(5)  int totalSegments;       // 总 .ts 片段数
    @HiveField(6)  int downloadedSegments;  // 已下载片段数
    @HiveField(7)  String localM3u8Path;    // 本地 M3U8 路径
    @HiveField(8)  String downloadDirectory;// 下载目录
    @HiveField(9)  String networkM3u8Url;   // 网络 M3U8 URL
    @HiveField(10) DateTime? completedAt;   // 完成时间
    @HiveField(11) String errorMessage;     // 错误信息
    @HiveField(12) int totalBytes;          // 总字节数
    @HiveField(13) String episodePageUrl;   // 剧集页面 URL
    @HiveField(14) String danmakuData;      // 缓存的弹幕数据（JSON）
    @HiveField(15) int danDanBangumiID;     // 弹弹Play 番剧 ID
}

class DownloadStatus {
    static const int pending = 0;       // 等待中
    static const int resolving = 1;     // 解析中
    static const int downloading = 2;   // 下载中
    static const int completed = 3;     // 已完成
    static const int failed = 4;        // 失败
    static const int paused = 5;        // 已暂停
}
```

## 4.12 Hive 注册流程

> 源码：`lib/hive_registrar.g.dart`（自动生成）

```dart
// 9 个 TypeAdapter 按 typeId 顺序注册
extension HiveRegistrar on HiveInterface {
    void registerAdapters() {
        registerAdapter(BangumiItemAdapter());       // typeId: 0
        registerAdapter(HistoryAdapter());           // typeId: 1
        registerAdapter(ProgressAdapter());          // typeId: 2
        registerAdapter(CollectedBangumiAdapter());  // typeId: 3
        registerAdapter(BangumiTagAdapter());        // typeId: 4
        registerAdapter(CollectedBangumiChangeAdapter()); // typeId: 5
        registerAdapter(SearchHistoryAdapter());     // typeId: 6
        registerAdapter(DownloadRecordAdapter());    // typeId: 7
        registerAdapter(DownloadEpisodeAdapter());   // typeId: 8
    }
}
```

在 `GStorage.init()` 中调用，确保所有类型在读写前注册完毕。

## 4.13 与 Java/Vue 项目的对比

| 概念 | Kazumi (Hive CE) | Spring Boot (JPA) | Vue (Vuex) |
|------|------------------|-------------------|------------|
| 数据库 | Hive（NoSQL 键值对） | MySQL/PostgreSQL（关系型） | localStorage/IndexedDB |
| ORM | TypeAdapter（自动生成） | Hibernate/JPA | 无 |
| 注解 | @HiveType + @HiveField | @Entity + @Column | 无 |
| 查询 | 仅按 key 查询 | SQL/JPQL | 手动过滤 |
| 迁移 | 需手动处理 | Flyway/Liquibase | 无 |
| 性能 | 极快（二进制） | 较快 | 较慢 |
| 关系 | 嵌套对象（无外键） | 外键 + JOIN | 嵌套对象 |

# 第10章 下载管理

> 下载系统负责把在线视频缓存到本地，实现离线播放。
> 核心流程：WebView 解析 M3U8 URL → 并发下载 .ts 片段 → 构建本地 M3U8 → mpv 离线播放。

---

## 10.1 完整链路

```
用户选择要下载的集数
    ↓
DownloadController.startDownload()
    ↓
┌─────────────────────────────────────────────────────────────────┐
│  第1步：创建下载记录 → Hive                                       │
│  状态: resolving                                                 │
└─────────────────────────────────────────────────────────────────┘
    ↓
┌─────────────────────────────────────────────────────────────────┐
│  第2步：WebView 解析 M3U8 URL                                     │
│  WebViewVideoSourceService.resolve(episodePageUrl)               │
│  → HeadlessWebView 加载剧集页面 → 拦截视频流                       │
│  状态: downloading                                               │
└─────────────────────────────────────────────────────────────────┘
    ↓
┌─────────────────────────────────────────────────────────────────┐
│  第3步：入队下载                                                  │
│  DownloadManager.enqueue() → 并行控制                            │
└─────────────────────────────────────────────────────────────────┘
    ↓
┌─────────────────────────────────────────────────────────────────┐
│  第4步：M3U8 下载引擎                                             │
│  4.1 获取 M3U8 内容（2MB 上限检查）                               │
│  4.2 解析 Master → 选最高清晰度                                   │
│  4.3 解析 Media → 获取 .ts 片段列表                               │
│  4.4 展开嵌套 M3U8（递归，最大 3 层）                              │
│  4.5 过滤广告片段                                                │
│  4.6 下载加密密钥                                                │
│  4.7 并发下载 .ts 片段（信号量控制 + 重试 3 次）                   │
│  4.8 构建本地 playlist.m3u8                                      │
│  状态: completed                                                 │
└─────────────────────────────────────────────────────────────────┘
    ↓
┌─────────────────────────────────────────────────────────────────┐
│  第5步：缓存弹幕（可选）                                          │
│  DanmakuApi.getDanDanmaku() → danmaku.json                      │
└─────────────────────────────────────────────────────────────────┘
    ↓
离线播放：mpv 加载本地 playlist.m3u8 → 播放本地 .ts 文件
```

## 10.3 数据模型

> 源码：`lib/modules/download/download_module.dart`

```dart
// 下载记录（一个番剧）
class DownloadRecord {
    int bangumiId;           // Bangumi ID
    String bangumiName;      // 番剧名
    String bangumiCover;     // 封面图
    String pluginName;       // 规则名（如 "AGE"）
    Map<int, DownloadEpisode> episodes;  // 集数 → 下载状态
    DateTime createdAt;      // 创建时间
    // key = "${pluginName}_${bangumiId}"
}

// 下载集数（一集）
class DownloadEpisode {
    int episodeNumber;       // 集数
    String episodeName;      // 集名
    int road;                // 播放源
    int status;              // 下载状态
    double progressPercent;  // 进度百分比
    int totalSegments;       // 总片段数
    int downloadedSegments;  // 已下载片段数
    String localM3u8Path;    // 本地 M3U8 路径（完成后设置）
    String downloadDirectory;// 下载目录
    String networkM3u8Url;   // 网络 M3U8 URL
    String errorMessage;     // 错误信息
    int totalBytes;          // 总字节数
    String episodePageUrl;   // 剧集页面 URL
    String danmakuData;      // 弹幕数据（已迁移到文件）
    int danDanBangumiID;     // 弹弹Play 番剧 ID
}

// 下载状态
class DownloadStatus {
    static const int pending = 0;      // 等待中（队列）
    static const int resolving = 1;    // 解析中（WebView 提取 M3U8 URL）
    static const int downloading = 2;  // 下载中
    static const int completed = 3;    // 已完成
    static const int failed = 4;       // 失败
    static const int paused = 5;       // 已暂停
}
```

### 状态机

```
resolving → downloading → completed
    ↓           ↓
  failed      failed
    ↓           ↓
  paused      paused → downloading（恢复）
    ↓
  retry → resolving（重新解析）
```

## 10.4 第1步：创建下载记录

> 源码：`lib/pages/download/download_controller.dart` → `startDownload()`

```dart
Future<void> startDownload({
    required int bangumiId,
    required String pluginName,
    required int episodeNumber,
    required String episodePageUrl,  // 剧集页面 URL（如 /play/12345-1）
    ...
}) async {
    final recordKey = '${pluginName}_$bangumiId';

    // 1. 获取或创建 DownloadRecord
    final record = _repository.getRecord(recordKey) ?? DownloadRecord(...);

    // 2. 去重检查（同一 URL 不重复下载）
    for (final entry in record.episodes.entries) {
        if (entry.value.episodePageUrl == episodePageUrl) return;
    }

    // 3. 创建 DownloadEpisode，状态设为 resolving
    final episode = DownloadEpisode(..., status: DownloadStatus.resolving);

    // 4. 写入 Hive
    record.episodes[episodeNumber] = episode;
    await _repository.putRecord(record);

    // 5. 加入解析队列
    _resolveQueue.add(_ResolveRequest(...));
    _processResolveQueue();
}
```

## 10.5 第2步：WebView 解析 M3U8 URL

> 源码：`lib/pages/download/download_controller.dart` → `_resolveAndEnqueue()`

```dart
Future<void> _resolveAndEnqueue(_ResolveRequest request) async {
    // 1. 找到对应规则
    final plugin = _findPlugin(request.pluginName);

    // 2. 拼接完整 URL
    final fullUrl = plugin.buildFullUrl(request.episodePageUrl);

    // 3. 用 WebView 解析视频 URL（和在线播放一样的流程）
    final videoSourceService = WebViewVideoSourceService();
    final source = await videoSourceService.resolve(
        fullUrl,
        useLegacyParser: plugin.useLegacyParser,
        timeout: Duration(seconds: 30),  // 下载比播放多等 15 秒
    );
    final m3u8Url = source.url;

    // 4. 更新状态为 downloading
    freshEpisode.networkM3u8Url = m3u8Url;
    freshEpisode.status = DownloadStatus.downloading;

    // 5. 构建 HTTP 头（User-Agent、Referer）
    final httpHeaders = plugin.buildHttpHeaders();

    // 6. 入队下载
    await _downloadManager.enqueue(DownloadRequest(
        m3u8Url: m3u8Url,
        httpHeaders: httpHeaders,
        adBlockerEnabled: true,
        ...
    ));

    // 7. 异步缓存弹幕（不阻塞下载）
    if (downloadDanmaku) {
        _fetchAndCacheDanmakuAsync(recordKey, bangumiId, episodeNumber);
    }
}
```

**关键：下载和在线播放用同一个 WebView 解析服务，区别是下载有更长的超时时间（30秒 vs 15秒）。**

## 10.6 第3步：入队与并行控制

> 源码：`lib/services/download/download_manager.dart` → `enqueue()`

```dart
Future<void> enqueue(DownloadRequest request) async {
    // 如果已经在下载 → 跳过
    if (_activeTasks.containsKey(key)) return;

    final task = DownloadTask(recordKey: request.recordKey, episodeNumber: request.episodeNumber);

    if (_runningCount < maxParallelEpisodes) {
        // 并行数未满 → 直接开始
        _runningCount++;
        _activeTasks[key] = task;
        _runEpisodeDownload(task: task, ...);
    } else {
        // 并行数已满 → 加入队列等待
        request.episode.status = DownloadStatus.pending;
        _queue.add(request);
        _activeTasks[key] = task;
    }
}
```

### 优先下载

```dart
Future<void> enqueuePriority(DownloadRequest request) async {
    // 从队列中移除 → 立即开始（绕过并行限制）
    _queue.removeWhere(...);
    _runningCount++;
    _runEpisodeDownload(task: task, ...);
}
```

### 并行设置

| 设置项 | 默认值 | 说明 |
|--------|--------|------|
| `maxParallelEpisodes` | 2 | 同时下载几集 |
| `maxParallelSegments` | 3 | 每集内同时下载几个 .ts 片段 |

用户可在设置页面调整（1-5 集，1-10 片段）。

## 10.7 ★ 第4步：M3U8 下载引擎

> 源码：`lib/services/download/download_manager.dart` → `_runEpisodeDownload()`

这是下载系统的核心，892 行代码中的精华。

### 4.1 获取 M3U8 内容

```dart
Future<String> _fetchM3u8(String url, Map<String, String> headers, CancelToken cancelToken) async {
    final content = await _http.getPlain(url, headers: headers,
        receiveTimeout: Duration(seconds: 15),
        onReceiveProgress: (received, total) {
            // 超过 2MB → 不是 M3U8，取消
            if (received > 2 * 1024 * 1024) fetchToken.cancel('too large');
        },
    );

    // 检查是否以 #EXTM3U 开头
    if (!content.trimLeft().startsWith('#EXTM3U')) {
        throw _NotM3u8Exception('URL 不是 M3U8 播放列表');
    }

    return content;
}
```

**如果 URL 不是 M3U8（比如直接是 .mp4），回退到直接文件下载。**

### 4.2 解析 M3U8

```dart
// 检测类型
final type = M3u8Parser.detectType(m3u8Content);

if (type == M3u8Type.master) {
    // Master 播放列表（多清晰度）→ 选最高带宽
    final master = M3u8Parser.parseMasterPlaylist(m3u8Content, m3u8Url);
    final bestVariant = master.bestVariant;
    mediaM3u8Content = await _fetchM3u8(bestVariant.uri, ...);
}

// 解析 Media 播放列表 → 获取 .ts 片段列表
final playlist = M3u8Parser.parseMediaPlaylist(mediaM3u8Content, mediaM3u8Url);
```

### 4.3 展开嵌套 M3U8

```dart
// 有些网站的 M3U8 里套着 M3U8（片段 URL 指向另一个 M3U8 文件）
final resolvedSegments = await M3u8Parser.resolveNestedSegments(
    playlist.segments,
    (url) => _fetchM3u8(url, headers, cancelToken),  // 递归获取
);
// 最大递归 3 层
```

### 4.4 过滤广告

```dart
List<M3u8Segment> segments = resolvedPlaylist.segments;
if (adBlockerEnabled) {
    segments = M3u8AdFilter.filterAds(segments);
}
```

广告识别启发式：
- 按 `discontinuityGroup` 分组
- 短片段组（< 30 秒）且在首尾 → 广告
- 单个片段 < 10 秒且与正片编码不同 → 广告

### 4.5 下载加密密钥

```dart
final keys = M3u8Parser.extractUniqueKeys(playlist);
final keyUriToLocal = <String, String>{};

for (int i = 0; i < keys.length; i++) {
    final keyFile = 'key_$i.key';
    final keyPath = '$episodeDir/$keyFile';
    await _downloadFile(keys[i].uri, keyPath, headers, cancelToken);
    keyUriToLocal[keys[i].uri] = keyFile;  // 远程 URI → 本地文件名
}
```

有些视频是 AES-128 加密的，需要先下载密钥文件。

### 4.6 并发下载 .ts 片段

```dart
// 信号量控制并发数
final semaphore = _Semaphore(maxParallelSegments);

// 跳过已下载的片段（断点续传）
final existingSegments = <int>{};
for (int i = 0; i < segments.length; i++) {
    final segFile = File('$episodeDir/seg_${i.toString().padLeft(5, '0')}.ts');
    if (await segFile.exists() && await segFile.length() > 0) {
        existingSegments.add(i);  // 已下载，跳过
    }
}

// 并发下载未完成的片段
for (final idx in pendingIndices) {
    await semaphore.acquire();  // 等待信号量

    _downloadSegmentWithRetry(
        segments[idx].uri,
        '$episodeDir/seg_${idx.toString().padLeft(5, '0')}.ts',
        headers,
        cancelToken,
    ).then((bytes) {
        episode.downloadedSegments++;
        episode.progressPercent = downloadedSegments / totalSegments;
        _speedTrackers[key]?.update(totalBytes);
        _notifyProgress(...);  // 通知 UI 刷新
        semaphore.release();
    });
}
```

### 4.7 重试机制

```dart
Future<int> _downloadSegmentWithRetry(String url, String savePath, ..., {int maxRetries = 3}) async {
    int retryCount = 0;
    while (true) {
        try {
            await _http.download(url, tmpPath, ...);
            await File(tmpPath).rename(savePath);  // 下载成功 → 重命名
            return await File(savePath).length();
        } catch (e) {
            retryCount++;
            if (retryCount >= maxRetries) rethrow;
            final delay = Duration(seconds: [1, 3, 9][retryCount - 1]);  // 指数退避
            await Future.delayed(delay);
        }
    }
}
```

## 10.8 直接文件下载（回退方案）

> 源码：`lib/services/download/download_manager.dart` → `_runDirectFileDownload()`

当 URL 不是 M3U8 时（比如直接是 .mp4），走直接下载：

```dart
Future<void> _runDirectFileDownload(...) async {
    // 支持断点续传（Range 请求头）
    if (existingBytes > 0) {
        requestHeaders['Range'] = 'bytes=$existingBytes-';
    }

    final response = await _http.getStream(videoUrl, headers: requestHeaders);

    // 流式写入文件
    final raf = await tmpFile.open(mode: FileMode.append);
    await for (final chunk in response.data!.stream) {
        await raf.writeFrom(chunk);
        received += chunk.length;
        episode.progressPercent = totalSize > 0 ? received / totalSize : 0;
        _notifyProgress(...);
    }

    await File(tmpPath).rename(filePath);  // 下载完成 → 重命名
    episode.localM3u8Path = filePath;
    episode.status = DownloadStatus.completed;
}
```

## 10.9 弹幕缓存

> 源码：`lib/pages/download/download_controller.dart` → `_fetchAndCacheDanmakuAsync()`

```dart
Future<void> _fetchAndCacheDanmakuAsync(recordKey, bangumiId, episodeNumber) async {
    // 1. 获取 DanDan 番剧 ID
    final danDanBangumiID = await DanmakuApi.getDanDanBangumiIDByBgmBangumiID(bangumiId);

    // 2. 获取弹幕列表
    final danmakus = await DanmakuApi.getDanDanmaku(danDanBangumiID, episodeNumber);

    // 3. 等待 downloadDirectory 就绪（下载管理器处理任务后才设置）
    String downloadDirectory = '';
    for (int i = 0; i < 10; i++) {
        final episode = _repository.getEpisode(...);
        if (episode.downloadDirectory.isNotEmpty) {
            downloadDirectory = episode.downloadDirectory;
            break;
        }
        await Future.delayed(Duration(seconds: 3));
    }

    // 4. 写入 danmaku.json
    await _writeDanmakuToFile(downloadDirectory, danmakus, danDanBangumiID);
}
```

## 10.11 存储优化

### 进度缓存

> 源码：`lib/repositories/download_repository.dart`

```dart
// 问题：每帧更新进度都写 Hive → I/O 太频繁 → 可能导致 Hive 损坏
// 解决：用内存缓存，只在状态变化时写入磁盘

final Map<String, DownloadEpisode> _progressCache = {};

Future<void> updateEpisode(String recordKey, int episodeNumber, DownloadEpisode episode) async {
    // 始终更新内存缓存
    _progressCache['${recordKey}_$episodeNumber'] = episode;

    // 只在状态变化时写入磁盘
    if (episode.status != previousStatus) {
        await _putToHive(recordKey, episodeNumber, episode);
    }
}
```

### 弹幕迁移

旧版本弹幕存在 Hive 里（`danmakuData` 字段），Hive compact 时会 OOM。新版本迁移到独立 JSON 文件：

```dart
// 启动时自动迁移
Future<void> _migrateDanmakuDataToFiles() async {
    for (final record in records) {
        for (final entry in record.episodes.entries) {
            if (episode.danmakuData.isNotEmpty) {
                // 写入独立文件
                await file.writeAsString('{"danDanBangumiID":...,"danmakus":...}');
                // 清空 Hive 字段
                episode.danmakuData = '';
            }
        }
    }
}
```

## 10.12 Android 后台服务

> 源码：`lib/services/download/background_download_service.dart`

Android 上 app 切到后台会被系统杀死。后台服务通过前台通知保活：

```dart
class BackgroundDownloadService {
    // 启动前台服务（显示通知栏）
    Future<bool> startService() async {
        await FlutterForegroundTask.startService(
            notificationTitle: 'Kazumi 正在下载',
            notificationText: '准备下载...',
        );
    }

    // 更新通知栏进度
    Future<void> updateProgress({
        required int activeCount,
        required int totalCount,
        required double overallProgress,
        required String speedText,
    }) async {
        await FlutterForegroundTask.updateService(
            notificationTitle: '下载中 ($activeCount/$totalCount)',
            notificationText: '${(overallProgress * 100).toInt()}% · $speedText',
        );
    }
}
```

**下载逻辑仍在主 isolate 中执行，后台服务只负责保活和通知栏 UI。**

## 10.14 关键设计总结

| 设计 | 说明 |
|------|------|
| 并行控制 | `_Semaphore` 控制每集内 .ts 片段的并发数（默认 3） |
| 集间并行 | `maxParallelEpisodes` 控制同时下载几集（默认 2） |
| 断点续传 | 已下载的 .ts 片段跳过，暂停后恢复不重新下载 |
| 重试机制 | 每个片段最多重试 3 次，指数退避（1s, 3s, 9s） |
| 存储检查 | 下载前检查可用空间（最低 100MB） |
| 广告过滤 | M3u8AdFilter 按 discontinuity group 识别并移除广告 |
| 弹幕缓存 | 下载时可选缓存弹幕到 danmaku.json，离线播放用 |
| 后台下载 | Android 前台服务保持进程存活，通知栏显示进度 |
| 进度优化 | _progressCache 避免每帧写 Hive，只在状态变化时持久化 |
| 弹幕迁移 | 旧版 Hive 弹幕数据自动迁移到独立 JSON 文件，防 OOM |
| M3U8 2MB 检查 | 获取 M3U8 时限制 2MB，超过认为不是 M3U8，回退直接下载 |

# 01 - 项目概览

## Kazumi 是什么

Kazumi 是一个跨平台动漫流媒体应用，支持 Windows、Android、iOS、macOS、Linux 五平台。它的核心能力是：**从任意动漫网站抓取视频流，带弹幕播放**。

### 它解决了什么问题

看番的痛点：不同动漫网站体验参差不齐，广告多、画质差、没有弹幕、不能跨设备同步。Kazumi 把这些问题一次性解决了：

```
用户视角：
  搜索番剧 → 选择源 → 在线播放（带弹幕）→ 下载离线看 → 跨设备同步收藏和进度

技术视角：
  规则引擎(XPath) 找到番剧 → WebView 拦截视频流 → mpv 播放器渲染 → 弹幕叠加显示
```

### 核心特性

| 特性 | 说明 |
|------|------|
| 规则引擎 | 用 JSON + XPath 描述网站结构，同一套代码爬不同网站 |
| 视频提取 | Headless WebView 加载播放页，拦截 M3U8/MP4 视频流 |
| 弹幕系统 | 聚合弹弹Play（B站等平台）弹幕，Canvas 实时渲染 |
| 硬件解码 | 支持 d3d11va/nvdec/mediacodec，GPU 解码不发热 |
| 超分辨率 | Anime4K GLSL 着色器，实时 2 倍画质提升 |
| 收藏同步 | WebDAV 双通道 + Bangumi 账号联动 |
| 离线下载 | M3U8 分片下载，本地 M3U8 播放列表 |
| 跨平台 | 一套 Dart 代码，五平台原生桥接 |

## 技术栈

| 类别 | 技术选型 | 说明 |
|------|----------|------|
| 框架 | Flutter 3.44.1, Dart 3.12.1 | 跨平台 UI 框架 |
| 路由/DI | flutter_modular | 模块化路由 + 依赖注入 |
| 状态管理 | MobX | 响应式状态（@observable + Observer） |
| 网络 | Dio (4 实例) | HTTP 客户端（API/下载/爬虫/GitHub） |
| 本地存储 | Hive CE | 轻量级 NoSQL（键值对，二进制序列化） |
| 视频播放 | media_kit (mpv fork) | 跨平台播放器（硬件解码 + 着色器） |
| 弹幕渲染 | canvas_danmaku | Canvas 绘制弹幕 |
| BBCode | ANTLR4 | Bangumi 社区富文本解析 |
| WebView | webview_windows / flutter_inappwebview | 视频提取 + 验证码处理 |
| 代码生成 | mobx_codegen, hive_ce_generator | .g.dart 自动生成 |

## 核心数据流

### 搜索到播放的完整链路

```
用户搜索 "进击的巨人"
    ↓
PluginSearchService.queryAllSource()
    ↓ 并发调用所有规则
Plugin.queryBangumi() × N  ← HTML + XPath 提取搜索结果
    ↓ Stream 逐个推送
InfoController.pluginSearchResponseList.add()
    ↓ MobX 响应式
SourceSheet 显示搜索结果 Tab（每个规则一个 Tab）
    ↓ 用户点击
VideoPageController.queryRoads()
    ↓ Plugin.querychapterRoads() ← HTML + XPath 提取剧集链接
WebViewVideoSourceService.resolve()
    ↓ HeadlessWebView 加载播放页 → 拦截 M3U8 URL
M3u8Parser.parseMediaPlaylist()
    ↓ 解析 .ts 片段列表
PlayerController.init()
    ↓ mpv 播放器加载 M3U8 → 硬件解码 → 渲染画面
    ↓ 同时：弹幕从弹弹Play API 加载 → Canvas 渲染
视频播放 + 弹幕显示
```

### 收藏同步链路

```
用户操作（添加/更新/删除收藏）
    ↓
CollectController → 记录 CollectedBangumiChange
    ↓
WebDAV: 下载远程 → mergeWebDav() → 回放本地 change → 上传
Bangumi: 拉取远程 → planBangumi() → 集合差异 + 优先级 → 四阶段执行
```

### 下载链路

```
用户选择集数
    ↓
DownloadController.startDownload()
    ↓ WebView 解析 M3U8 URL
DownloadManager._runEpisodeDownload()
    ↓ 获取 M3U8 → 解析片段 → 过滤广告 → 并发下载 .ts
    ↓ 构建本地 playlist.m3u8
离线播放：mpv 加载本地 M3U8
```

## 核心设计模式

| 模式 | 应用场景 | 说明 |
|------|----------|------|
| Singleton | BangumiClient、WebDav、GStorage | 全局唯一实例 |
| Factory | VideoWebviewControllerFactory | 平台 WebView 创建 |
| Observer | MobX @observable + Observer Widget | 响应式 UI 更新 |
| Repository | IHistoryRepository → HistoryRepository | 接口隔离 + Hive 实现 |
| Interceptor | BangumiMirrorInterceptor、GithubMirrorInterceptor | 请求拦截 + URL 重写 |
| Composition | PlayerController 组合 6 个子控制器 | 播放器功能拆分 |
| Event Sourcing | HistorySyncEvent + HistorySyncMerger | 观看历史多端同步 |
| Change Log | CollectedBangumiChange | 收藏变更日志同步 |

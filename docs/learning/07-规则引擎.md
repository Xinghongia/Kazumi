# 第5章 规则引擎 —— 项目的核心灵魂

> Kazumi 的目的就是看番，而规则引擎决定了它能从哪些网站抓到番剧资源。
> 整套系统用 **JSON + XPath** 描述任意动漫网站的 HTML 结构，让同一套代码能爬不同的网站。

---

## 5.1 规则长什么样

一个规则就是一个 JSON 文件。以 `assets/plugins/AGE.json` 为例：

```json
{
    "api": "1",
    "type": "anime",
    "name": "AGE",
    "version": "1.5",
    "baseURL": "https://www.agedm.io/",
    "searchURL": "https://www.agedm.io/search?query=@keyword",
    "searchList": "//div[2]/div/section/div/div/div/div",
    "searchName": "//div/div[2]/h5/a",
    "searchResult": "//div/div[2]/h5/a",
    "chapterRoads": "//div[2]/div/section/div/div[2]/div[2]/div[2]/div",
    "chapterResult": "//ul/li/a",
    "muliSources": true,
    "useWebview": true,
    "useNativePlayer": true
}
```

### 字段速查表

| 字段 | 作用 | 举例 |
|------|------|------|
| `name` | 规则名称（唯一标识） | `"AGE"` |
| `baseURL` | 目标网站根地址 | `"https://www.agedm.io/"` |
| `searchURL` | 搜索 URL 模板，`@keyword` 是占位符 | `"https://xxx/search?query=@keyword"` |
| `searchList` | XPath：搜索结果列表容器 | `"//div[2]/div/section/div/div/div/div"` |
| `searchName` | XPath：从卡片中提取番剧名 | `"//div/div[2]/h5/a"` |
| `searchResult` | XPath：从卡片中提取详情页链接 | `"//div/div[2]/h5/a"` |
| `chapterRoads` | XPath：剧集列表容器 | `"//div[2]/div/section/..."` |
| `chapterResult` | XPath：每集链接 | `"//ul/li/a"` |
| `usePost` | 搜索是否用 POST 请求 | `false`（默认 GET） |
| `useWebview` | 是否需要 WebView 加载 | `true` |
| `antiCrawlerConfig` | 反爬/验证码配置 | 见下文 |

### XPath 是什么

XPath 就是用路径表达式从 HTML/XML 中定位节点。类似 CSS 选择器，但更强大。

```
假设 HTML 是这样的：

<div class="results">              ← //div[@class="results"]
  <div class="card">               ← //div[@class="results"]/div
    <h5>
      <a href="/bangumi/123">       ← //div[@class="results"]/div/h5/a
        进击的巨人
      </a>
    </h5>
  </div>
</div>
```

在规则里：
- `searchList` 定位到所有 `.card` 容器
- `searchName` 从容器里取文字（番剧名）
- `searchResult` 从容器里取 `href` 属性（链接）

## 5.2 规则存储

### 内置规则

项目源码中 `assets/plugins/` 目录下有 3 个内置规则：

```
assets/plugins/
  ├── AGE.json     ← agedm.io（AGE动漫）
  ├── DM84.json    ← dm84.com
  └── aafun.json   ← aafun.me
```

### 运行时存储

所有规则（内置 + 用户添加 + 仓库下载）都存在一个文件里：

```
Windows:  %APPDATA%\com.example\kazumi\plugins\v2\plugins.json
Android:  /data/data/<package>/app_flutter/plugins/v2/plugins.json
```

`plugins.json` 是一个 JSON 数组，每个元素就是一个规则对象。

### 远程规则仓库

更多规则在 GitHub 仓库 `Predidit/KazumiRules` 中，用户通过"规则仓库"页面在线安装。

## 5.4 规则引擎核心：Plugin 类

> 源码：`lib/plugins/plugins.dart`

一个 `Plugin` 实例既是数据模型，又是规则引擎。

### 搜索方法：queryBangumi(keyword)

```dart
Future<PluginSearchResponse> queryBangumi(String keyword) async {
    // 1. 构造搜索 URL
    String queryURL = searchURL.replaceAll('@keyword', Uri.encodeQueryComponent(keyword));

    // 2. 发送 HTTP 请求获取 HTML
    String htmlString = await _siteClient.getText(queryURL, headers: httpHeaders);

    // 3. 解析 HTML 为 DOM 树
    var htmlElement = parse(htmlString).documentElement!;

    // 4. 反爬检测
    if (detectsCaptchaChallenge(htmlString)) {
        throw CaptchaRequiredException(name);
    }

    // 5. 用 XPath 提取搜索结果
    htmlElement.queryXPath(searchList).nodes.forEach((element) {
        SearchItem searchItem = SearchItem(
            name: element.queryXPath(searchName).node!.text,      // 番剧名
            src: element.queryXPath(searchResult).node!['href'],   // 详情页链接
        );
        searchItems.add(searchItem);
    });

    return PluginSearchResponse(pluginName: name, data: searchItems);
}
```

**流程图：**

```
用户输入 "进击的巨人"
    ↓
searchURL.replace('@keyword', '进击的巨人')
→ "https://www.agedm.io/search?query=进击的巨人"
    ↓
PluginSiteClient.getText(url) → 拿到 HTML 字符串
    ↓
parse(htmlString) → 解析成 DOM 树
    ↓
detectsCaptchaChallenge() → 是验证码页面？→ 抛出 CaptchaRequiredException
    ↓
htmlElement.queryXPath(searchList) → 找到所有搜索结果卡片
    ↓
遍历每个卡片：
  - queryXPath(searchName) → 提取番剧名
  - queryXPath(searchResult) → 提取详情页链接 (href)
    ↓
返回 PluginSearchResponse(pluginName: "AGE", data: [SearchItem...])
```

### 剧集提取方法：querychapterRoads(url)

```dart
Future<List<Road>> querychapterRoads(String url) async {
    // 1. 请求详情页 HTML
    String htmlString = await _siteClient.getText(queryURL, headers: httpHeaders);

    // 2. 解析 HTML
    var htmlElement = parse(htmlString).documentElement!;

    // 3. 用 chapterRoads XPath 找到所有播放源容器
    htmlElement.queryXPath(chapterRoads).nodes.forEach((element) {
        // 4. 用 chapterResult XPath 提取每集的链接和名字
        element.queryXPath(chapterResult).nodes.forEach((item) {
            chapterUrlList.add(item.node.attributes['href']);
            chapterNameList.add(item.node.text);
        });

        // 5. 组装成 Road 对象
        roadList.add(Road(name: "播放列表1", data: chapterUrlList, identifier: chapterNameList));
    });

    return roadList;
}
```

### 反爬检测：detectsCaptchaChallenge()

三种检测方式（由 `AntiCrawlerConfig` 配置）：

| 方式 | 说明 |
|------|------|
| XPath | 用 XPath 在 HTML 中查找验证码元素 |
| Text | 直接在 HTML 字符串中查找纯文本 |
| Regex | 用正则表达式匹配 |

## 5.5 规则管理器：PluginsController

> 源码：`lib/plugins/plugins_controller.dart`

### 初始化流程

```
init()
  ↓
创建目录：plugins/v2/
  ↓
loadAllPlugins()
  ↓
读取 plugins/v2/plugins.json → JSON 字符串 → List<Plugin> → pluginList
```

### 存储兼容

```
旧版本（< 1.5.1）：每个规则一个 JSON 文件（AGE.json、DM84.json...）
新版本（≥ 1.5.1）：所有规则合并为一个 plugins.json

loadAllPlugins() 会自动检测：
  - plugins.json 存在 → 直接读取
  - 不存在 → 从旧版单文件迁移 → 合并 → 删除旧文件
```

### CRUD 操作

```dart
// 更新或新增（所有添加规则的入口最终都调用这个方法）
void updatePlugin(Plugin plugin) {
    // 有同名规则 → 替换
    // 没有 → 追加
    pluginList.add(plugin);
    savePlugins();  // → 写入 plugins.json
}

// 删除
void removePlugin(Plugin plugin) { ... }

// 拖拽排序
void onReorder(int oldIndex, int newIndex) { ... }

// 持久化
void savePlugins() {
    final jsonData = jsonEncode(pluginListToJson());
    pluginsFile.writeAsString(jsonData);
}
```

### 远程更新

```
queryPluginHTTPList() → 从 GitHub 拉规则仓库列表
pluginStatus() → 对比本地版本和远程版本
  - 'installed': 已装最新
  - 'update': 可更新
  - 'install': 未安装
tryUpdatePlugin() → 下载远程规则 → updatePlugin() → savePlugins()
```

## 5.6 搜索调度器：PluginSearchService

> 源码：`lib/services/plugin/plugin_search_service.dart`

### queryAllSource(keyword) — 并发搜索所有规则

```dart
Future<void> queryAllSource(String keyword) async {
    // 1. 所有规则状态设为 'pending'
    for (Plugin plugin in pluginsController.pluginList) {
        infoController.pluginSearchStatus[plugin.name] = 'pending';
    }

    // 2. 并发发起所有搜索请求（fire-and-forget，不等待）
    for (Plugin plugin in pluginsController.pluginList) {
        plugin.queryBangumi(keyword).then((result) {
            infoController.pluginSearchStatus[plugin.name] = 'success';
            _controller?.add(result);  // 通过 Stream 推送
        }).catchError((error) {
            if (error is CaptchaRequiredException) → 状态: 'captcha'
            if (error is NoResultException) → 状态: 'noResult'
            else → 状态: 'error'
        });
    }

    // 3. 监听 Stream，逐个收到结果 → add 到 pluginSearchResponseList
    await for (var result in _controller!.stream) {
        infoController.pluginSearchResponseList.add(result);
    }
}
```

**关键设计：不是 await 等待所有结果，而是用 Stream 逐个推送，哪个先回来就先显示。**

### 错误处理

| 异常类型 | 状态 | UI 表现 |
|----------|------|---------|
| `CaptchaRequiredException` | `'captcha'` | 显示"需要验证码" + "进行验证"按钮 |
| `NoResultException` | `'noResult'` | 显示"无结果" + "别名检索"/"手动检索" |
| `SearchErrorException` | `'error'` | 显示"检索失败" + "重试"按钮 |
| 正常返回 | `'success'` | 显示搜索结果卡片列表 |

### 点击搜索结果 → 播放

```dart
onTap: () async {
    // 1. 设置播放页控制器上下文
    videoPageController.bangumiItem = infoController.bangumiItem;
    videoPageController.currentPlugin = plugin;
    videoPageController.title = searchItem.name;

```

### 验证码处理

三种验证码类型：

| 类型 | 处理方式 |
|------|----------|
| `imageCaptcha` | 弹出对话框，显示验证码图片，用户手动输入 |
| `autoClickButton` | 自动检测验证按钮并模拟点击 |
| `customJavaScript` | 执行规则中配置的自定义 JS 脚本 |

验证码通过后：
1. 保存 Cookie 到 `PluginCookieManager`
2. 重新搜索该规则（带 Cookie）
3. 后续请求自动带上 Cookie

## 5.9 与 Java/Vue 项目的对比

| 概念 | Kazumi (Flutter) | Spring Boot | Vue |
|------|------------------|-------------|-----|
| 规则定义 | JSON + XPath | 通常是代码硬编码 | 配置文件 |
| 规则引擎 | Plugin 类（XPath 解析） | Jsoup / Selenium | Puppeteer / Cheerio |
| 规则管理 | PluginsController (MobX) | @Service + 数据库 | Vuex/Pinia Store |
| 搜索调度 | PluginSearchService + Stream | CompletableFuture | Promise.all |
| 结果展示 | Observer + TabBar | Thymeleaf / Vue | v-for + 组件 |
| 反爬处理 | WebView + Cookie | HttpClient + Proxy | Puppeteer + Stealth |

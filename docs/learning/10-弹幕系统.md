# 第8章 弹幕系统

> 弹幕系统比较简单：调用弹弹Play API 获取弹幕列表，按秒分组，播放时实时渲染。
> 弹弹Play 是一个弹幕聚合平台，从 B 站、Gamer 等平台收集弹幕。

---

## 8.1 完整链路

```
VideoPageController._loadPlaybackDanmaku()
    ↓
PlayerDanmakuController.fetchDanmaku(bangumiId, pluginName, episode)
    ↓
┌──────────────────────────────────────────────────────────────┐
│  第1步：Bangumi ID → 弹弹Play ID                              │

    ↓

```

## 8.3 弹弹Play API

> 源码：`lib/request/apis/danmaku_api.dart`

弹弹Play 是一个弹幕聚合平台，从 B 站、Gamer 等平台收集弹幕。Kazumi 通过它的 API 获取弹幕数据。

### API 地址

| API | 地址 | 用途 |
|-----|------|------|
| 番剧查询 | `/api/v2/bangumi/bgmtv/{bangumiId}` | Bangumi ID → 弹弹Play ID |
| 弹幕列表 | `/api/v2/comment/{bangumiID}{episode}` | 获取某集所有弹幕 |
| 搜索 | `/api/v2/search/anime` | 按标题搜索番剧 |
| 番剧详情 | `/api/v2/bangumi/{bangumiID}` | 获取番剧分集信息 |

### API 凭证

> 源码：`lib/utils/dandan_credentials.dart`

```dart
const Map<String, String> dandanCredentials = {
    'id': String.fromEnvironment('DANDANAPI_APPID'),     // 编译时注入
    'value': String.fromEnvironment('DANDANAPI_KEY'),     // 编译时注入
};
```

弹弹Play API 需要 AppID + Key 签名认证，编译时通过 `--dart-define` 注入。本地跑源码没有这个 key，弹幕功能不可用。

## 8.4 第1步：Bangumi ID → 弹弹Play ID

```dart
// DanmakuApi.getDanDanBangumiIDByBgmBangumiID(bangumiId)
//   → 调用 /api/v2/bangumi/bgmtv/{bangumiId}
//   → 返回弹弹Play的番剧ID

static Future<int> getDanDanBangumiIDByBgmBangumiID(int bgmBangumiID) async {
    var path = ApiEndpoints.formatUrl(
        ApiEndpoints.dandanAPIInfoByBgmBangumiId, [bgmBangumiID]);
    var endPoint = ApiEndpoints.dandanAPIDomain + path;
    final jsonData = await _client.get(endPoint);
    DanmakuEpisodeResponse response = DanmakuEpisodeResponse.fromJson(jsonData);
    return response.bangumiId;
}
```

Kazumi 用的是 Bangumi 的番剧 ID，弹弹Play 有自己的 ID 体系，所以需要先转换。

## 8.5 第2步：获取弹幕列表

```dart
// DanmakuApi.getDanDanmaku(bangumiID, episode)
//   → 调用 /api/v2/comment/{bangumiID}{episode}
//   → 返回该集的所有弹幕

static Future<List<DanmakuEntry>> getDanDanmaku(int bangumiID, int episode) async {
    // 弹弹Play 的弹幕ID规则：番剧ID + 集数（4位补零）
    // 例如：番剧ID=1758，第1集 → 弹幕ID=17580001
    var path = ApiEndpoints.dandanAPIComment +
        bangumiID.toString() +
        episode.toString().padLeft(4, '0');
    var endPoint = ApiEndpoints.dandanAPIDomain + path;

    final jsonData = await _client.get(endPoint, queryParameters: {'withRelated': 'true'});
    List<dynamic> comments = jsonData['comments'];

    List<DanmakuEntry> danmakus = [];
    for (var comment in comments) {
        danmakus.add(DanmakuEntry.fromJson(comment));
    }
    return danmakus;
}
```

### 弹幕 JSON 格式

弹弹Play 返回的弹幕格式：

```json
{
    "comments": [
        {
            "m": "前方高能",
            "p": "12.5,1,16777215,[BiliBili]"
        },
        {
            "m": "太好看了",
            "p": "12.8,1,16711680,[BiliBili]"
        }
    ]
}
```

- `m`：弹幕内容
- `p`：参数，逗号分隔：`时间,类型,颜色,来源`

## 8.6 弹幕数据模型

> 源码：`lib/modules/danmaku/danmaku_module.dart`

```dart
class DanmakuEntry {
    String message;  // 弹幕内容，如 "前方高能"
    double time;     // 出现时间（秒），如 12.5
    int type;        // 类型：1=滚动，4=底部，5=顶部
    Color color;     // 颜色
    String source;   // 来源："[BiliBili]"、"[Gamer]"

    factory DanmakuEntry.fromJson(Map<String, dynamic> json) {
        String messageValue = json['m'];
        List<String> parts = json['p'].split(',');
        double timeValue = double.parse(parts[0]);      // 时间
        int typeValue = int.parse(parts[1]);             // 类型
        Color color = generateDanmakuColor(int.parse(parts[2]));  // 颜色
        String sourceValue = parts[3];                   // 来源
        return DanmakuEntry(...);
    }
}
```

## 8.7 第3步：按秒分组

> 源码：`lib/pages/player/controller/player_danmaku_controller.dart` → `addDanmakus()`

```dart
void addDanmakus(List<DanmakuEntry> danmakus) {
    // 可选：去重（5秒内相同内容的弹幕只保留一条）
    final List<DanmakuEntry> listToAdd = danmakuDeduplicationEnable
        ? mergeDuplicateDanmakus(danmakus, timeWindowSeconds: 5)
        : danmakus;

    // 按秒分组存入 Map
    for (var element in listToAdd) {
        var danmakuList = danDanmakus[element.time.toInt()] ?? List.empty(growable: true);
        danmakuList.add(element);
        danDanmakus[element.time.toInt()] = danmakuList;
    }
}
```

分组后的数据结构：

```dart
Map<int, List<DanmakuEntry>> danDanmakus = {
    0:  [DanmakuEntry("开始啦", 0.5), DanmakuEntry("来了来了", 0.8)],
    5:  [DanmakuEntry("好帅", 5.2)],
    12: [DanmakuEntry("前方高能", 12.5), DanmakuEntry("注意", 12.8)],
    // ...
};
```

这样播放到第 N 秒时，直接 `danDanmakus[N]` 就能取出该秒的所有弹幕。

## 8.8 第4步：播放时实时渲染

弹幕渲染使用 `canvas_danmaku` 包，基于 Canvas 绘制。

```dart
// 获取当前播放时间对应的弹幕
List<DanmakuEntry> danmakusForPlaybackPosition(Duration playbackPosition) {
    final danmakuSecond = resolveDanmakuSecond(playbackPosition);
    return danDanmakus[danmakuSecond] ?? const [];
}

// canvas_danmaku 包内部：
// 每帧检查当前播放时间 → 取出弹幕 → Canvas 绘制滚动文字
```

### 渲染流程

```
播放器当前时间 = 12.5秒
    ↓
resolveDanmakuSecond(12.5秒) → 12
    ↓
danDanmakus[12] → [前方高能, 注意]
    ↓
canvas_danmaku 渲染：
  - 前方高能 → 从右向左滚动
  - 注意 → 从右向左滚动（不同轨道）
```

## 8.9 弹幕时间偏移

弹幕时间和视频时间可能不完全对齐（片头曲长度不同等），Kazumi 支持手动偏移：

```dart
double get timelineOffsetSeconds {
    final offset = GStorage.getSetting(SettingsKeys.danmakuTimeOffset);
    return offset;
}

int? resolveDanmakuSecond(Duration playbackPosition) {
    final sourceMilliseconds = playbackPosition.inMilliseconds -
        (timelineOffsetSeconds * 1000).round();
    return Duration(milliseconds: sourceMilliseconds).inSeconds;
}
```

用户可以在设置中调整偏移量（正负秒数）。

## 8.10 弹幕来源

弹弹Play 是一个弹幕聚合平台，弹幕来源包括：

| 来源 | 标记 | 说明 |
|------|------|------|
| B 站 | `[BiliBili]` | 最主要的弹幕来源 |
| Gamer | `[Gamer]` | 游戏相关弹幕 |
| 其他 | `[其他]` | 其他平台 |

**Kazumi 不自己产生弹幕，所有弹幕都是从弹弹Play API 获取的（本质上是从 B 站搬运的）。**

## 8.11 本地缓存

> 源码：`lib/pages/player/controller/player_danmaku_controller.dart` → `_fetchCachedDanmaku()`

本地播放时有缓存逻辑：

```dart
Future<DanmakuLoadResult> _fetchCachedDanmaku(int bangumiId, String pluginName, int episode) async {
    // 1. 先查本地缓存
    final cachedDanmakus = await downloadController.getCachedDanmakus(
        bangumiId, pluginName, episode);

    if (cachedDanmakus != null && cachedDanmakus.isNotEmpty) {
        // 有缓存 → 直接用
        return DanmakuLoadResult.success(danmakus: cachedDanmakus, ...);
    } else {
        // 没缓存 → 请求网络
        var res = await DanmakuApi.getDanDanmaku(nextBangumiID, episode);
        // 请求完自动缓存到本地
        _saveDanmakuToCache(downloadController, bangumiId, pluginName, episode, res, nextBangumiID);
        return DanmakuLoadResult.success(danmakus: res, ...);
    }
}
```

## 8.14 总结

弹幕系统的核心链路：

```
Bangumi ID
    ↓ /api/v2/bangumi/bgmtv/{id}
弹弹Play ID
    ↓ /api/v2/comment/{id}{episode}
弹幕列表 (List<DanmakuEntry>)
    ↓ 按秒分组
Map<int, List<DanmakuEntry>>
    ↓ 播放时实时取出
canvas_danmaku 渲染
```

**简单、直接、不复杂。** 唯一需要注意的是弹幕来源是弹弹Play（聚合了 B 站等平台的弹幕），不是 Kazumi 自己产生的。

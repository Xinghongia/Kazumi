# 第12章 超分辨率与着色器

> Kazumi 支持基于 Anime4K 的实时超分辨率，通过 GPU 着色器在播放时提升画质。
> Anime4K 使用 CNN（卷积神经网络）结构的着色器，将神经网络权重编码为纹理采样，
> 在 GPU 上并行执行推理，比 CPU 推理快数十倍，适合实时视频处理。

---

## 12.1 完整链路

```
应用启动
    ↓
InitPage._loadShaders()
    → ShaderAssetService.copyShadersToExternalDirectory()
    → 扫描 assets/shaders/*.glsl → 复制到 应用目录/anime_shaders/
    ↓
播放器创建
    ↓
PlayerPlaybackController.createVideoController()
    → 读取 defaultSuperResolutionType 设置
    → 如果不是 OFF → setShader(type)
    → 如果是 mediacodec_embed → 强制 OFF（不支持着色器）
    ↓
着色器应用
    ↓
setShader(type)
    → type=1: mpv change-list glsl-shaders clr "" (清空)
    → type=2: mpv change-list glsl-shaders set <lite路径>
    → type=3: mpv change-list glsl-shaders set <full路径>
    ↓
mpv 渲染流水线
    原始帧 → Clamp_Highlights → Restore_CNN → Upscale_CNN_x2 → AutoDownscalePre → 输出
```

## 12.3 着色器文件说明

> 目录：`assets/shaders/`

| 文件 | 作用 | 大小 | 模型 |
|------|------|------|------|
| `Anime4K_Clamp_Highlights.glsl` | 钳制高光值，防止上采样伪影 | 3KB | - |
| `Anime4K_Restore_CNN_S.glsl` | 画面修复 + 降噪 | 17KB | Small |
| `Anime4K_Restore_CNN_M.glsl` | 画面修复 + 降噪 | 36KB | Medium |
| `Anime4K_Restore_CNN_VL.glsl` | 画面修复 + 降噪 | 145KB | Very Large |
| `Anime4K_Upscale_CNN_x2_S.glsl` | 2倍超分辨率放大 | 19KB | Small |
| `Anime4K_Upscale_CNN_x2_M.glsl` | 2倍超分辨率放大 | 38KB | Medium |
| `Anime4K_Upscale_CNN_x2_VL.glsl` | 2倍超分辨率放大 | 148KB | Very Large |
| `Anime4K_AutoDownscalePre_x2.glsl` | 自动降采样 2x（防止过度放大） | 2KB | - |
| `Anime4K_AutoDownscalePre_x4.glsl` | 自动降采样 4x | 2KB | - |

### 三类着色器

```
1. 画面修复（Restore_CNN）
   → 降噪、去伪影、修复压缩瑕疵
   → S/M/VL 三个模型，越大越清晰但越耗 GPU

2. 超分辨率放大（Upscale_CNN_x2）
   → 2倍放大（720p → 1440p）
   → S/M/VL 三个模型

3. 辅助着色器
   → Clamp_Highlights：钳制高光，防止后续处理产生色块
   → AutoDownscalePre：检测是否过度放大，自动降回
```

## 12.5 着色器应用（mpv 集成）

> 源码：`lib/pages/player/controller/player_playback_controller.dart` → `setShader()`

```dart
Future<void> setShader(int type, {Player? player}) async {
    var pp = player.platform as NativePlayer;
    await pp.waitForPlayerInitialization;
    await pp.waitForVideoControllerInitializationIfAttached;

    if (type == 2) {
        // 效率档：加载 Lite 着色器管线
        await pp.command([
            'change-list',
            'glsl-shaders',
            'set',
            buildShadersAbsolutePath(
                shaderAssetService.shadersDirectory.path,
                mpvAnime4KShadersLite,
            ),
        ]);
        superResolutionType = 2;
        return;
    }

    if (type == 3) {
        // 质量档：加载 Full 着色器管线
        await pp.command([
            'change-list',
            'glsl-shaders',
            'set',
            buildShadersAbsolutePath(
                shaderAssetService.shadersDirectory.path,
                mpvAnime4KShaders,
            ),
        ]);
        superResolutionType = 3;
        return;
    }

    // 关闭：清空所有着色器
    await pp.command(['change-list', 'glsl-shaders', 'clr', '']);
    superResolutionType = 1;
}
```

### mpv 命令说明

```
change-list glsl-shaders set <路径列表>   → 设置着色器列表（替换）
change-list glsl-shaders clr ""          → 清空所有着色器
```

路径格式：
- Windows：`path1;path2;path3`（分号分隔）
- Linux/macOS：`path1:path2:path3`（冒号分隔）

## 12.6 着色器路径构建

> 源码：`lib/utils/media.dart`

```dart
String buildShadersAbsolutePath(String baseDirectory, List<String> shaders) {
    final absolutePaths = shaders.map((shader) {
        return path.join(baseDirectory, shader);
    }).toList();

    // Windows 用 ; 分隔，其他平台用 : 分隔（mpv 要求）
    if (Platform.isWindows) {
        return absolutePaths.join(';');
    }
    return absolutePaths.join(':');
}

// 输出示例（Windows）：
// "C:\Users\xxx\...\anime_shaders\Anime4K_Clamp_Highlights.glsl;
//  C:\Users\xxx\...\anime_shaders\Anime4K_Restore_CNN_VL.glsl;
//  C:\Users\xxx\...\anime_shaders\Anime4K_Upscale_CNN_x2_VL.glsl;..."
```

## 12.7 着色器加载服务

> 源码：`lib/services/shaders/shader_asset_service.dart`

```dart
class ShaderAssetService {
    late Directory shadersDirectory;  // 着色器目录

    Future<void> copyShadersToExternalDirectory() async {
        // 1. 扫描 assets 中的所有 .glsl 文件
        final assets = await AssetManifest.loadFromAssetBundle(rootBundle);
        final shaderFiles = assets.where((a) =>
            a.startsWith('assets/shaders/') && a.endsWith('.glsl'));

        // 2. 创建目标目录
        shadersDirectory = Directory('${appSupportDir.path}/anime_shaders');
        if (!await shadersDirectory.exists()) {
            await shadersDirectory.create(recursive: true);
        }

        // 3. 复制着色器文件（已存在则跳过）
        for (var filePath in shaderFiles) {
            final targetFile = File('${shadersDirectory.path}/$fileName');
            if (await targetFile.exists()) continue;  // 跳过

            final data = await rootBundle.load(filePath);
            await targetFile.writeAsBytes(data.buffer.asUint8List());
        }
    }
}
```

**为什么需要复制？** Flutter 的 asset bundle 是只读的，mpv 需要文件系统路径才能加载着色器。所以启动时把 .glsl 从 asset bundle 复制到应用支持目录。

## 12.8 GLSL 着色器结构

以 `Anime4K_Clamp_Highlights.glsl` 为例：

```glsl
//!DESC Anime4K-v4.0-De-Ring-Compute-Statistics  ← 着色器描述
//!HOOK MAIN                                      ← 挂钩点：主渲染流程
//!BIND HOOKED                                    ← 绑定输入纹理
//!SAVE STATSMAX                                  ← 输出到中间纹理
//!COMPONENTS 1                                   ← 单通道输出

#define KERNELSIZE 5          // 卷积核大小

float get_luma(vec4 rgba) {
    return dot(vec4(0.299, 0.587, 0.114, 0.0), rgba);  // RGB → 亮度
}

vec4 hook() {
    float gmax = 0.0;
    for (int i=0; i<KERNELSIZE; i++) {
        float g = get_luma(MAIN_texOff(vec2(i - 2, 0)));
        gmax = max(g, gmax);  // 5x1 卷积核取最大亮度
    }
    return vec4(gmax, 0.0, 0.0, 0.0);
}
```

### mpv 着色器指令

| 指令 | 说明 |
|------|------|
| `//!DESC` | 着色器描述（调试用） |
| `//!HOOK` | 挂钩点：MAIN（主渲染）、PREKERNEL（内核前）等 |
| `//!BIND` | 绑定输入纹理 |
| `//!SAVE` | 输出纹理名（中间结果可被后续着色器引用） |
| `//!COMPONENTS` | 输出通道数（1=灰度，4=RGBA） |

### 渲染流水线

```
原始视频帧
    ↓
//!HOOK MAIN
Clamp_Highlights（第1个 pass：计算行最大亮度）
    → SAVE STATSMAX
    ↓
//!HOOK MAIN + BIND STATSMAX
Clamp_Highlights（第2个 pass：计算列最大亮度）
    → SAVE STATSMAX
    ↓
//!HOOK PREKERNEL + BIND STATSMAX
Clamp_Highlights（第3个 pass：钳制高光）
    → 输出到下一阶段
    ↓
Restore_CNN（画面修复）
    ↓
Upscale_CNN_x2（2倍放大）
    ↓
AutoDownscalePre（自动降采样）
    ↓
最终输出画面
```

## 12.9 CNN 超分辨率原理

### 传统插值 vs CNN

```
传统插值放大（双线性/双三次）：
  像素 → 数学公式 → 插值填充 → 模糊的放大图
  优点：快
  缺点：模糊，丢失细节

Anime4K CNN 放大：
  像素 → 卷积神经网络 → 学习到的特征提取 → 清晰的放大图
  优点：清晰，保留细节
  缺点：GPU 负载高
```

### 神经网络权重编码

```
普通神经推理（PyTorch/TensorFlow）：
  模型文件 → CPU/GPU 加载权重 → 矩阵运算 → 输出

Anime4K 着色器方式：
  GLSL 文件 → 权重编码为 const 数组/纹理采样 → GPU 并行卷积 → 输出

优势：
  - 不需要 ML 框架
  - 利用 GPU 纹理单元并行计算
  - 实时处理视频帧（60fps+）
  - 跨平台（任何支持 OpenGL/Vulkan 的 GPU）
```

### CNN 结构（以 Restore_CNN 为例）

```
输入帧（RGB）
    ↓
卷积层 1（3x3 卷积 + ReLU）← 权重在 GLSL 的 const 数组中
    ↓
卷积层 2（3x3 卷积 + ReLU）
    ↓
卷积层 3（3x3 卷积）
    ↓
残差连接（输入 + 输出）
    ↓
修复后的帧
```

**每个卷积层是一个 `hook()` 函数，权重硬编码在着色器代码中。S/M/VL 模型的区别是层数和通道数不同。**

```dart
void handleSuperResolutionChange(int shaderIndex) {
    // 1. mediacodec_embed 不支持着色器
    if (videoRenderer == 'mediacodec_embed') {
        showDialog('此渲染器不支持超分辨率');
        return;
    }

    // 2. 质量档需要性能警告
    if (shaderIndex == 3 && showWarning) {
        showDialog('质量档需要较好的显卡，可能导致卡顿');
        // 带"不再提示"复选框
    }

    // 3. 应用着色器
    playerController.setShader(shaderIndex);
}
```

## 12.11 与硬件解码的关系

```dart
// mediandec_embed 渲染器不支持着色器
if (videoRenderer == 'mediacodec_embed') {
    hAenable = true;
    hardwareDecoder = 'mediacodec';
    superResolutionType = 1;  // 强制关闭超分辨率
}
```

| 渲染器 | 硬件解码 | 超分辨率 |
|--------|----------|----------|
| gpu (OpenGL) | 支持 | 支持 |
| gpu-next (Vulkan) | 支持 | 支持 |
| mediandec_embed | 支持 | **不支持** |

**mediandec_embed 是 Android 原生 MediaCodec 直通渲染，不经过 mpv 的着色器管线，所以无法使用超分辨率。**

## 12.12 性能影响

| 模式 | GPU 负载 | 适用场景 |
|------|----------|----------|
| 关闭 | 无 | 低端显卡 / 电池模式 |
| 效率档 | 中等 | 中端显卡 / 日常使用 |
| 质量档 | 很高 | 高端显卡 / 追求画质 |

### 为什么质量档需要警告

```
效率档：~117KB 着色器，7个 pass
  → 大多数独立显卡和中端手机 GPU 可以流畅运行

质量档：~338KB 着色器，6个 pass（但 VL 模型大 10 倍）
  → Restore_CNN_VL 单个着色器 145KB，卷积层数多
  → 需要较强的 GPU 才能保持 60fps
  → 集成显卡/低端手机可能掉帧
```

## 12.13 与 Java/Vue 项目的对比

| 概念 | Kazumi (Flutter) | 传统方案 |
|------|------------------|----------|
| 超分辨率 | Anime4K GLSL 着色器 | ffmpeg + Real-ESRGAN |
| 执行位置 | GPU 实时（播放时） | CPU/GPU 离线（预处理） |
| 处理速度 | 实时 60fps | 每帧数秒 |
| 模型格式 | GLSL const 数组 | ONNX/PyTorch 权重文件 |
| 集成方式 | mpv 着色器管线 | 独立处理工具 |
| 质量 | 中等（实时限制） | 最高（无时间限制） |

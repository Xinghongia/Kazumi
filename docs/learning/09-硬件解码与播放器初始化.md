# 第7章 硬件解码与播放器初始化

> 上一章我们拿到了视频 URL（M3U8），这一章讲 mpv 播放器怎么把它变成画面。
> 核心问题是：为什么默认很烫，以及硬件解码如何解决这个问题。

---

## 7.1 从视频 URL 到画面播放的完整链路

```
VideoPageController._resolveWithVideoSourceService()
    ↓ 拿到 VideoSource(url: "https://cdn.xxx/episode1.m3u8")
PlayerController.init(PlaybackInitParams)
    ↓
PlayerPlaybackController.createVideoController(httpHeaders, adBlockerEnabled)
    ↓
┌─────────────────────────────────────────────────────────────────┐
│  第1步：读取用户设置                                              │
│  第2步：创建 mpv Player 实例                                     │
│  第3步：配置 mpv 底层参数                                         │
│  第4步：Android 选择视频渲染器                                    │
│  ★ 第5步：创建 VideoController（硬件解码在这里生效）              │
│  第6步：打开视频并播放                                            │
└─────────────────────────────────────────────────────────────────┘
    ↓
mpv 内部：
  1. 解析 M3U8 → 获取 .ts 片段列表
  2. 下载 .ts 片段
  3. 解码视频帧（hwdec='auto' → GPU, hwdec='no' → CPU）
  4. 渲染到屏幕
```

## 7.3 第1步：读取用户设置

```dart
Future<Player?> createVideoController(
    Map<String, String> httpHeaders, bool adBlockerEnabled,
    {int offset = 0}) async {

    // 从本地存储读取用户设置
    hAenable = GStorage.getSetting(SettingsKeys.hAenable);           // 硬件解码开关（默认 false）
    hardwareDecoder = GStorage.getSetting(SettingsKeys.hardwareDecoder); // 解码器类型（默认 'auto'）
    autoPlay = GStorage.getSetting(SettingsKeys.autoPlay);           // 自动播放
    lowMemoryMode = GStorage.getSetting(SettingsKeys.lowMemoryMode); // 低内存模式
    superResolutionType = GStorage.getSetting(SettingsKeys.defaultSuperResolutionType); // 超分辨率
    ...
}
```

**关键：`hAenable` 默认是 `false`，这就是默认用软件解码的原因。**

## 7.4 第2步：创建 mpv Player 实例

```dart
final Player player = Player(
    configuration: PlayerConfiguration(
        bufferSize: lowMemoryMode ? 15 * 1024 * 1024 : 1500 * 1024 * 1024,
        //           ↑ 低内存模式: 15MB              ↑ 正常模式: 1.5GB
        osc: false,           // 不显示 mpv 默认的屏幕控制界面
        logLevel: MPVLogLevel.values[debug.playerLogLevel],
        adBlocker: adBlockerEnabled,  // 广告过滤
    ),
);
mediaPlayer = player;
```

`Player` 是 media_kit 包提供的播放器类，内部封装了 mpv。

## 7.5 第3步：配置 mpv 底层参数

mpv 是一个命令行播放器，所有配置都通过 `setProperty` 设置：

```dart
// 获取 mpv 原生播放器实例
var pp = player.platform as NativePlayer;

// 磁盘缓存目录（mpv 默认按 Linux 路径配置，其他平台会损坏，这里修正）
await pp.setProperty("demuxer-cache-dir", await getPlayerTempPath());

// 音频处理：时间拉伸算法（变速播放时保持音调不变）
await pp.setProperty("af", "scaletempo2=max-speed=8");

// 代理设置（如果用户配置了）
if (proxyEnable) {
    await pp.setProperty("http-proxy", formattedProxy);
}

// Android 音频输出
if (Platform.isAndroid) {
    await pp.setProperty("volume-max", "100");
    if (androidEnableOpenSLES) {
        await pp.setProperty("ao", "opensles");   // OpenSL ES（低延迟）
    } else {
        await pp.setProperty("ao", "audiotrack");  // AudioTrack（兼容性好）
    }
}
```

## 7.6 第4步：Android 视频渲染器选择

```dart
String? videoRenderer;
if (Platform.isAndroid) {
    final String androidVideoRenderer =
        GStorage.getSetting(SettingsKeys.androidVideoRenderer);

    if (androidVideoRenderer == 'auto') {
        // Android 14+ 使用 gpu-next（基于 Vulkan，着色器性能更好）
        // Android 14 以下使用 gpu（基于 OpenGL）
        final int androidSdkVersion = await PlatformEnvironmentService.getAndroidSdkVersion();
        if (androidSdkVersion >= 34) {
            videoRenderer = 'gpu-next';
        } else {
            videoRenderer = 'gpu';
        }
    } else {
        videoRenderer = androidVideoRenderer;
    }
}

// 特殊情况：如果选择了 mediacodec_embed 渲染器
// 强制开启硬件解码，使用 Android 原生 MediaCodec
if (videoRenderer == 'mediacodec_embed') {
    hAenable = true;
    hardwareDecoder = 'mediacodec';
    superResolutionType = 1;  // MediaCodec 不支持超分辨率，强制关闭
}
```

### Android 渲染器选项

| 渲染器 | 说明 | 适用 |
|--------|------|------|
| `auto` | 自动选择 | 推荐 |
| `gpu` | OpenGL 渲染 | Android 14 以下 |
| `gpu-next` | Vulkan 渲染 | Android 14+ |
| `mediacodec_embed` | Android 原生 MediaCodec | 兼容性最好 |

## 7.7 ★ 第5步：硬件解码配置（核心）

```dart
videoController ??= VideoController(
    player,
    configuration: VideoControllerConfiguration(
        vo: videoRenderer,                                // 视频渲染器
        enableHardwareAcceleration: hAenable,             // ★ 硬件加速总开关
        hwdec: hAenable ? hardwareDecoder : 'no',         // ★ 解码器选择
        androidAttachSurfaceAfterVideoParameters: false,
    ),
);
```

### 这一行代码决定了一切

```dart
hwdec: hAenable ? hardwareDecoder : 'no'
```

| hAenable | hardwareDecoder | hwdec 值 | 效果 |
|----------|-----------------|----------|------|
| `false` (默认) | 任意 | `'no'` | **CPU 软件解码**（FFmpeg，发热严重） |
| `true` | `'auto'` | `'auto'` | 自动选最佳硬件解码器 |
| `true` | `'d3d11va'` | `'d3d11va'` | DirectX 11 硬件解码（Windows） |
| `true` | `'nvdec'` | `'nvdec'` | NVIDIA 硬件解码 |
| `true` | `'mediacodec'` | `'mediacodec'` | Android 原生硬件解码 |

## 7.8 第6步：打开视频并播放

```dart
await player.open(
    Media(videoUrl(),                                    // 视频 URL（M3U8 或 MP4）
        start: Duration(seconds: offset),               // 续播位置
        httpHeaders: httpHeaders),                       // HTTP 头（User-Agent、Referer）
    play: autoPlay,                                      // 是否自动播放
);
```

`player.open()` 之后，mpv 内部开始工作：

```
Media("https://cdn.xxx/episode1.m3u8")
    ↓
mpv 解析 URL，识别为 HLS 流
    ↓
下载 M3U8 Master 播放列表 → 选择最高带宽变体
    ↓
下载 M3U8 Media 播放列表 → 获取 .ts 片段列表
    ↓
循环：下载 .ts 片段 → 解码 → 渲染
    │
    ├─ hwdec = 'no'   → FFmpeg CPU 解码 → 发热
    └─ hwdec = 'auto' → GPU 硬件解码 → 不发热
```

## 7.9 软件解码 vs 硬件解码

```
软件解码（默认，hwdec='no'）：

  .ts 片段
      ↓
  CPU (FFmpeg 软件解码)      ← CPU 满载，发热严重
      ↓
  解码后的视频帧
      ↓
  GPU 渲染到屏幕
      ↓
  画面显示


硬件解码（开启后，hwdec='auto'）：

  .ts 片段
      ↓
  GPU/专用芯片 (d3d11va/nvdec/mediacodec)  ← CPU 几乎不参与，不发热
      ↓
  解码后的视频帧
      ↓
  GPU 渲染到屏幕
      ↓
  画面显示
```

### 为什么 B 站 4K 不烫

| 对比项 | B 站 | Kazumi (默认) | Kazumi (开硬件解码) |
|--------|------|---------------|---------------------|
| 解码方式 | MediaCodec 硬件解码 | FFmpeg 软件解码 | d3d11va/nvdec 硬件解码 |
| CPU 占用 | 极低 | 很高 | 极低 |
| 发热 | 不烫 | 烫 | 不烫 |
| 兼容性 | 好（官方优化） | 最好（通用） | 可能有兼容问题 |

**B 站用的是 Android/iOS 系统级硬件解码 API，和 Kazumi 开启硬件解码后一样。**

## 7.10 可选硬件解码器列表

> 源码：`lib/utils/constants.dart`

```dart
const Map<String, String> hardwareDecodersList = {
    'auto':         '启用任意可用解码器',
    'auto-safe':    '启用最佳解码器',
    'auto-copy':    '启用带拷贝功能的最佳解码器',
    'd3d11va':      'DirectX11 (Windows 8+)',
    'd3d11va-copy': 'DirectX11 (Windows 8+) (非直通)',
    'videotoolbox': 'VideoToolbox (macOS / iOS)',
    'vaapi':        'VAAPI (Linux)',
    'nvdec':        'NVDEC (NVIDIA 独占)',
    'nvdec-copy':   'NVDEC (NVIDIA 独占) (非直通)',
    'vulkan':       'Vulkan (全平台) (实验性)',
    'dxva2':        'DXVA2 (Windows 7+)',
    'mediacodec':   'MediaCodec (Android)',
};
```

### 选择建议

| 平台 | 推荐解码器 | 说明 |
|------|-----------|------|
| Windows | `auto` 或 `d3d11va` | 自动选最佳，或手动指定 DirectX 11 |
| Windows (N卡) | `nvdec` | NVIDIA 显卡专用，效率最高 |
| Android | `auto` 或 `mediacodec` | 系统原生解码 |
| macOS/iOS | `videotoolbox` | Apple 原生解码 |
| Linux | `vaapi` 或 `nvdec` | 取决于显卡驱动 |

### -copy 后缀的区别

- **无 copy**（如 `d3d11va`）：解码后的帧直接在 GPU 显存中，零拷贝，性能最好
- **有 copy**（如 `d3d11va-copy`）：解码后的帧从 GPU 拷贝到 CPU 内存，兼容性更好但性能略差

## 7.11 超分辨率

> 源码：`lib/pages/player/controller/player_playback_controller.dart` → `setShader()`

Kazumi 支持 Anime4K 超分辨率，通过 mpv 的 GLSL 着色器实现：

```dart
Future<void> setShader(int type, {Player? player}) async {
    var pp = player.platform as NativePlayer;

    if (type == 2) {
        // Anime4K Efficiency（性能优先）
        await pp.command([
            'change-list', 'glsl-shaders', 'set',
            buildShadersAbsolutePath(shadersDirectory, mpvAnime4KShadersLite),
        ]);
    }
    if (type == 3) {
        // Anime4K Quality（质量优先）
        await pp.command([
            'change-list', 'glsl-shaders', 'set',
            buildShadersAbsolutePath(shadersDirectory, mpvAnime4KShaders),
        ]);
    }
}
```

**注意：超分辨率需要开启硬件解码才能生效，且渲染器必须是 `gpu` 或 `gpu-next`。**

## 7.15 总结

### 默认发烫的原因

```dart
hAenable = false  // 默认关闭硬件解码
    ↓
hwdec = 'no'      // 告诉 mpv 不要用硬件解码
    ↓
FFmpeg CPU 软件解码  // 所有解码工作压在 CPU 上
    ↓
CPU 满载 → 发热 → 卡顿
```

### 解决方法

设置 → 播放设置 → 硬件解码 → **打开**

```dart
hAenable = true
    ↓
hwdec = 'auto'    // mpv 自动选最佳硬件解码器
    ↓
GPU/专用芯片解码  // CPU 几乎不参与
    ↓
不发热 → 流畅 → 省电
```

### 作者默认关闭的原因

硬件解码在某些设备上兼容性不好，可能导致：
- 黑屏（解码器不支持该视频格式）
- 花屏（解码器实现有 bug）
- 崩溃（驱动问题）

所以作者选择了最安全的默认值（软件解码），让用户自己选择是否开启。

# 02 - 启动流程

## 入口文件：main.dart

```dart
void main() async {
  // ============================================================
  // 第 1 步：Flutter 绑定初始化
  // ============================================================
  // 为什么必须第一个调用？
  // - main() 是 async 函数，Dart 的 async 需要 Binding 来处理平台通道 (Platform Channel)
  // - WidgetsFlutterBinding.ensureInitialized() 做两件事：
  //   1. 创建 WidgetsFlutterBinding 单例（如果还没有）
  //   2. 初始化 Flutter 引擎和平台通道
  // - 没有这一步，后续所有涉及平台通道的异步操作都会报错：
  //   Hive、MediaKit、path_provider、windowManager 等全部依赖平台通道
  WidgetsFlutterBinding.ensureInitialized();

  // ============================================================
  // 第 2 步：媒体库初始化
  // ============================================================
  // MediaKit 是视频播放引擎（基于 mpv），需要在使用前初始化
  // 内部会注册平台相关的解码器和渲染器
  MediaKit.ensureInitialized();

  // ============================================================
  // 第 3 步：[Android/iOS] 沉浸式状态栏
  // ============================================================
  // edgeToEdge 模式：状态栏和导航栏透明，内容延伸到屏幕边缘
  // 这是现代 Android/iOS 的推荐做法，配合 SafeArea 使用
  if (Platform.isAndroid || Platform.isIOS) {
    SystemChrome.setEnabledSystemUIMode(SystemUiMode.edgeToEdge);
    SystemChrome.setSystemUIOverlayStyle(const SystemUiOverlayStyle(
      systemNavigationBarColor: Colors.transparent,
      systemNavigationBarDividerColor: Colors.transparent,
      statusBarColor: Colors.transparent,
    ));
  }

  // ============================================================
  // 第 4 步：[Android] WebView 特性检测
  // ============================================================
  // 检测 Android 系统 WebView 版本是否支持高级特性
  // 如 DocumentStartScript（页面加载前注入 JS）
  // 只在 Android 上需要，其他平台跳过
  if (Platform.isAndroid) {
    await WebViewFeatureService.initialize();
  }

  // ============================================================
  // 第 5 步：本地数据库初始化（带优雅降级）
  // ============================================================
  // 初始化链：path_provider → Hive → GStorage
  //
  // path_provider: 获取应用专属存储目录（各平台不同）
  //   - Windows: C:\Users\xxx\AppData\Roaming\kazumi
  //   - Android: /data/data/com.predidit.kazumi
  //   - macOS: ~/Library/Application Support/kazumi
  //
  // Hive: 高性能 NoSQL 二进制数据库，Flutter 生态最常用的本地存储
  //   - initFlutter() 会创建 hive 目录并初始化引擎
  //
  // GStorage: 项目自定义的存储管理器，封装了 8 个 Hive Box
  //   - favorites, collectibles, histories, settings 等
  //   - 注册所有 TypeAdapter（数据模型的序列化器）
  try {
    final hivePath = '${(await getApplicationSupportDirectory()).path}/hive';
    await Hive.initFlutter(hivePath);
    await GStorage.init();
  } catch (e) {
    // 优雅降级：存储初始化失败时，不崩溃，显示专门的错误页面
    // 这是一个很好的用户体验设计：告诉用户出了什么问题，而不是白屏
    debugPrint('Storage initialization failed: $e');

    // 桌面端也需要初始化窗口管理器，否则窗口不会显示
    if (isDesktop()) {
      await windowManager.ensureInitialized();
      windowManager.waitUntilReadyToShow(null, () async {
        await windowManager.show();
        await windowManager.focus();
      });
    }

    // 显示错误页面，然后直接 return，不执行后续初始化
    runApp(MaterialApp(
        title: '初始化失败',
        localizationsDelegates: GlobalMaterialLocalizations.delegates,
        supportedLocales: const [
          Locale.fromSubtags(
              languageCode: 'zh', scriptCode: 'Hans', countryCode: "CN")
        ],
        locale: const Locale.fromSubtags(
            languageCode: 'zh', scriptCode: 'Hans', countryCode: "CN"),
        builder: (context, child) {
          return const StorageErrorPage();
        }));
    return;  // 关键：直接返回，不执行后续任何初始化
  }

  // ============================================================
  // 第 6 步：[Desktop] 窗口管理器配置
  // ============================================================
  // window_manager 是桌面端窗口管理库，控制：
  // - 窗口大小（低分辨率设备用小窗口）
  // - 标题栏样式（macOS 始终隐藏，Windows 可配置）
  // - 窗口按钮可见性（最小化/最大化/关闭）
  bool showWindowButton =
      await GStorage.getSetting(SettingsKeys.showWindowButton);

  if (isDesktop()) {
    await windowManager.ensureInitialized();
    final lowResolution = await isLowResolution();
    WindowOptions windowOptions = WindowOptions(
      // 低分辨率设备用小窗口，否则用默认大小
      size: lowResolution ? const Size(840, 600) : const Size(1280, 860),
      center: true,
      skipTaskbar: false,
      // macOS 始终隐藏标题栏（使用自定义标题栏）
      // Windows 根据用户设置决定
      titleBarStyle: (Platform.isMacOS || !showWindowButton)
          ? TitleBarStyle.hidden
          : TitleBarStyle.normal,
      windowButtonVisibility: showWindowButton,
      title: 'Kazumi',
    );
    // waitUntilReadyToShow: 等窗口准备好了再显示，避免闪烁
    windowManager.waitUntilReadyToShow(windowOptions, () async {
      await windowManager.show();
      await windowManager.focus();
    });
  }

  // ============================================================
  // 第 7 步：代理设置
  // ============================================================
  // 从 GStorage 读取用户配置的代理设置，应用到 HttpClient
  // 影响后续所有 HTTP 请求（包括 Dio 和图片加载）
  ProxyManager.applyProxy();

  // ============================================================
  // 第 8 步：启动应用
  // ============================================================
  // 三层嵌套，各司其职：
  //
  // ChangeNotifierProvider<ThemeProvider>
  //   → 提供主题状态（亮色/暗色/跟随系统，主题色，字体）
  //   → 使用 Provider 包，因为主题只需要简单的 ChangeNotifier
  //
  // ModularApp<AppModule>
  //   → 提供依赖注入 (DI) 和路由管理
  //   → 使用 flutter_modular 包，因为需要模块化的 DI 和路由
  //
  // AppWidget
  //   → 根 Widget，负责 MaterialApp.router、托盘、窗口生命周期
  runApp(
    ChangeNotifierProvider(
      create: (_) => ThemeProvider(),
      child: ModularApp(
        module: AppModule(),
        child: const AppWidget(),
      ),
    ),
  );
}
```

## 初始化顺序的依赖关系

每一步都依赖前一步的结果，所以必须用 `await` 串行执行：

```
WidgetsFlutterBinding.ensureInitialized()
    │
    │  创建 Binding → 后续平台通道才能工作
    │
    ▼
MediaKit.ensureInitialized()
    │
    │  需要平台通道来注册解码器
    │
    ▼
Hive.initFlutter(hivePath)
    │
    │  需要 path_provider（依赖平台通道）获取存储目录
    │
    ▼
GStorage.init()
    │
    │  需要 Hive 就绪才能打开 Box
    │
    ▼
windowManager.ensureInitialized()
    │
    │  需要平台通道来操作原生窗口
    │
    ▼
ProxyManager.applyProxy()
    │
    │  需要 GStorage 读取代理配置
    │
    ▼
runApp(...)  ← 最后才启动 UI，确保所有基础设施就绪
```

## 从 main.dart 到首页的完整调用链

main.dart 只是"基础设施初始化"，真正的"应用初始化"在 InitPage 中完成：

```
main.dart (入口 — 基础设施初始化)
    │
    │  runApp(
    │    ChangeNotifierProvider<ThemeProvider>
    │      └─ ModularApp<AppModule>
    │           └─ AppWidget
    │  )
    │
    ▼
AppWidget (根 Widget)
    │
    │  build() → MaterialApp.router(
    │    routerConfig: Modular.routerConfig,  ← Modular 接管路由
    │  )
    │
    │  Modular 根据 AppModule 路由表匹配到 "/"
    │
    ▼
IndexModule (路由中枢)
    │
    │  routes: r.child("/", child: (_) => InitPage())
    │
    ▼
InitPage (启动页 — 应用初始化)
    │
    │  _initializeApp() 按顺序执行：
    │    ├─ 1. _migrateStorage()         数据迁移（旧版本兼容）
    │    ├─ 2. _loadShaders()            复制超分辨率着色器
    │    ├─ 3. _loadDanmakuShield()      加载弹幕屏蔽列表
    │    ├─ 4. _webDavInit()             WebDAV 同步（如果启用）
    │    ├─ 5. _bangumiInit()            Bangumi 同步（如果启用）
    │    ├─ 6. downloadController.init() 下载管理器初始化
    │    ├─ 7. _checkRunningOnX11()      [Linux] X11 环境检测
    │    ├─ 8. _pluginInit()             ★ 插件初始化 + 免责声明
    │    ├─ 9. _showShortcutDialog()     [Windows] 桌面快捷方式
    │    ├─ 10. _startDefaultPage()      ★ 跳转到默认首页
    │    └─ 11. _update()                检查应用更新
    │
    │  _startDefaultPage() {
    │    Modular.to.navigate("/tab/popular/");
    │  }
    │
    ▼
IndexPage (导航壳)
    │
    │  ScaffoldMenu → PageView → RouterOutlet
    │
    ▼
PopularPage (推荐页 — 用户看到的第一个页面)
```

### 三个阶段

| 阶段 | 文件 | 做什么 |
|------|------|--------|
| **基础设施** | `main.dart` | Flutter 引擎、播放器、数据库、窗口 |
| **应用初始化** | `init_page.dart` | 插件、同步、着色器、免责声明 |
| **UI 渲染** | `index_page.dart` → `popular_page.dart` | 导航壳 + 首页 |

### InitPage 的初始化流程

`init_page.dart` 是应用真正"活起来"的地方。它的 `build()` 只返回一个空的 `Scaffold`，不做任何 UI 渲染，唯一职责是执行初始化序列，然后跳转到真正的首页。

```dart
Future<void> _initializeApp() async {
  _migrateStorage();           // 1. 旧版本数据迁移
  _loadShaders();              // 2. 复制 Anime4K 着色器到外部目录
  _loadDanmakuShield();        // 3. 加载弹幕屏蔽词列表
  _webDavInit();               // 4. [可选] WebDAV 连接 + 同步历史
  _bangumiInit();              // 5. [可选] Bangumi Token 验证 + 同步
  downloadController.init();   // 6. 下载管理器就绪
  _checkRunningOnX11();        // 7. [Linux] X11 环境检测
  _pluginInit();               // 8. ★ 加载插件规则 + 免责声明
  _showShortcutDialog();       // 9. [Windows] 桌面快捷方式
  _startDefaultPage();         // 10. ★ 跳转到首页
  _update();                   // 11. 检查应用更新
}
```

### _pluginInit() — 插件初始化（最复杂的一步）

```dart
Future<void> _pluginInit() async {
  // 加载插件规则文件
  await pluginsController.init();

  // 如果插件列表为空（首次安装），弹出免责声明
  if (pluginsController.pluginList.isEmpty) {
    await KazumiDialog.show(builder: (context) {
      return AlertDialog(
        title: const Text('免责声明'),
        content: Text(statementsText),
        actions: [
          TextButton(onPressed: () => exit(0), child: Text('退出')),
          TextButton(onPressed: () async {
            // 用户同意后，复制内置插件到外部目录
            await pluginsController.copyPluginsToExternalDirectory();
          }, child: Text('已阅读并同意')),
        ],
      );
    });
  }
}
```

### _startDefaultPage() — 跳转到首页

```dart
void _startDefaultPage() {
  // 从 GStorage 读取用户设置的默认启动页（通常是 "/tab/popular/"）
  final defaultStartupPage = GStorage.getSetting(SettingsKeys.defaultStartupPage);

  // 设置动态主题色（避免暗色模式下白屏闪烁）
  themeProvider.setDynamic(GStorage.getSetting(SettingsKeys.useDynamicColor));

  // 路由跳转到首页
  Modular.to.navigate(defaultStartupPage);
}
```

## 关键设计点

### 1. 优雅降级

```dart
try {
  await GStorage.init();
} catch (e) {
  // 不崩溃，显示专门的错误页面
  runApp(MaterialApp(builder: (_) => const StorageErrorPage()));
  return;  // 直接返回，不执行后续初始化
}
```

存储初始化失败时，应用不会白屏或崩溃，而是显示一个友好的错误页面。

### 2. 双重状态管理

```dart
runApp(
  ChangeNotifierProvider<ThemeProvider>  // Provider 管主题
    child: ModularApp<AppModule>         // Modular 管 DI + 路由
      child: AppWidget
)
```

- **Provider**：只负责主题状态 (ThemeProvider)，简单轻量
- **Modular**：负责依赖注入和路由管理，功能强大

两者各司其职，不互相干扰。

### 3. 平台分支

```dart
if (isDesktop()) {
  // 桌面端：窗口管理器、标题栏、托盘
  await windowManager.ensureInitialized();
} else {
  // 移动端：沉浸式状态栏
  SystemChrome.setEnabledSystemUIMode(SystemUiMode.edgeToEdge);
}
```

同一个 `main()` 函数，根据平台执行不同的初始化逻辑。

### 4. 窗口关闭行为

AppWidget 中处理了三种窗口关闭策略：

```dart
void onWindowClose() {
  final exitBehavior = GStorage.getSetting(SettingsKeys.exitBehavior);

  switch (exitBehavior) {
    case 0: exit(0);              // 直接退出
    case 1: windowManager.hide(); // 最小化到托盘
    default:                      // 弹窗询问
      KazumiDialog.show(builder: (_) => AlertDialog(
        // 退出 / 最小化 / 取消
      ));
  }
}
```

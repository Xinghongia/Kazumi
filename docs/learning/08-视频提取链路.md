# 第6章 视频提取链路 —— 从搜索结果到视频播放

> 规则引擎解决了"找到番"的问题，视频提取链路解决"播放番"的问题。
> 本章从用户点击搜索结果那一刻开始，追踪到视频真正播放出来的完整过程。

---

## 6.1 与规则引擎的衔接点

在 `source_sheet.dart` 中，用户点击搜索结果卡片时触发：

```dart
onTap: () async {
    // 规则引擎产出的数据
    videoPageController.bangumiItem = infoController.bangumiItem;  // 番剧信息
    videoPageController.currentPlugin = plugin;                     // 当前规则
    videoPageController.title = searchItem.name;                    // 搜索结果标题
    videoPageController.src = searchItem.src;                       // 详情页 URL（规则引擎产出）

    // ★ 衔接点：把规则引擎产出的 URL 传给视频提取链路
    await videoPageController.queryRoads(searchItem.src, plugin.name);

    // 跳转播放页
    Modular.to.pushNamed('/video/');
}
```

**规则引擎产出的 `searchItem.src`（详情页 URL）就是视频提取链路的输入。**

## 6.4 第一步：获取剧集列表

> 源码：`lib/pages/video/video_controller.dart` → `queryRoads()`

```dart
Future<void> queryRoads(String url, String pluginName) async {
    // 找到对应的规则
    for (Plugin plugin in pluginsController.pluginList) {
        if (plugin.name == pluginName) {
            // 调用规则引擎的剧集提取方法（XPath 解析详情页 HTML）
            roadList.addAll(await plugin.querychapterRoads(url));
        }
    }
}
```

### Road 数据结构

```dart
class Road {
    String name;           // 播放源名称，如 "播放列表1"
    List<String> data;     // 每集的播放页 URL，如 ["/play/12345-1", "/play/12345-2"]
    List<String> identifier; // 每集的名字，如 ["第1集", "第2集"]
}
```

一个番剧可能有多个播放源（不同线路），每个播放源有多个剧集。

### 与规则引擎的关系

```
规则引擎 Plugin.querychapterRoads(url)
    ↓
用 chapterRoads XPath 定位播放源容器
    ↓
用 chapterResult XPath 提取每集链接
    ↓
组装成 Road 对象列表
    ↓
VideoPageController.roadList（MobX 可观察列表）
    ↓
播放页 UI 显示剧集列表供用户选择
```

## 6.5 第二步：提取视频流 URL（核心）

> 源码：`lib/services/video_source/webview_video_source_service.dart`

这是整个链路最关键的部分。很多动漫网站的视频地址是通过 JS 动态生成的，直接 HTTP 请求拿不到。所以用 **Headless WebView**（无头浏览器）加载页面，让网站自己执行 JS，然后拦截网络请求捕获视频地址。

### 流程

```dart
Future<VideoSource> resolve(String episodeUrl, ...) async {
    // 1. 创建平台对应的 WebView 控制器
    _webview = VideoWebviewControllerFactory.getController();
    await _webview!.init();

    // 2. 用 WebView 加载剧集页面
    await _webview!.loadUrl(episodeUrl, useLegacyParser, offset: offset);

    // 3. 等待 WebView 拦截到视频流 URL
    final event = await _webview!.onVideoURLParser.first.timeout(
        Duration(seconds: 15),
        onTimeout: () => throw VideoSourceTimeoutException(),
    );

    // 4. 返回视频源
    return VideoSource(url: event.$1, offset: event.$2);
}
```

### WebView 拦截原理

```
HeadlessWebView.loadUrl("https://www.agedm.io/play/12345-1")
    ↓
WebView 加载页面（执行 JS、渲染 DOM）
    ↓
页面中的 <video> 标签或 JS 脚本发起视频请求：
  - .m3u8 请求 → onM3USourceLoaded 事件
  - .mp4 请求  → onVideoSourceLoaded 事件
    ↓
WebView 拦截到这些请求的 URL
    ↓
通过 videoParserEventController.add((url, offset)) 推送出去
    ↓
WebViewVideoSourceService.resolve() 收到 → 返回 VideoSource
```

### 平台工厂模式

```dart
class VideoWebviewControllerFactory {
    static VideoWebviewController getController() {
        if (Platform.isWindows)  → VideoWebviewWindowsImpl  (webview_windows)
        if (Platform.isLinux)    → VideoWebviewLinuxImpl
        if (Platform.isMacOS)    → VideoWebviewAppleImpl
        if (Platform.isAndroid)  → VideoWebviewAndroidImpl  (flutter_inappwebview)
        默认                     → VideoWebviewImpl
    }
}
```

### Windows 实现详解

> 源码：`lib/webview/video/impl/video_webview_windows_impl.dart`

```dart
class VideoWebviewWindowsImpl extends VideoWebviewController<WebviewController> {
    HeadlessWebview? headlessWebview;  // 无头 WebView

    Future<void> init() async {
        await _setupProxy();                    // 配置代理（如果用户设置了）
        headlessWebview = HeadlessWebview();    // 创建无头 WebView
        await headlessWebview!.run();           // 启动
    }

    Future<void> loadUrl(String url, ...) async {
        // 监听 M3U8 请求拦截事件
        subscriptions.add(headlessWebview!.onM3USourceLoaded.listen((data) {
            String url = data['url'];
            videoParserEventController.add((url, offset));  // 推送视频 URL
        }));

        // 监听普通视频请求拦截事件
        subscriptions.add(headlessWebview!.onVideoSourceLoaded.listen((data) {
            String url = data['url'];
            videoParserEventController.add((url, offset));  // 推送视频 URL
        }));

        // 加载剧集页面
        await headlessWebview!.loadUrl(url);
    }
}
```

**关键点：WebView 不是用来展示给用户看的，而是用来"骗"网站执行 JS、生成视频地址的。**

## 6.6 第三步：解析 M3U8 播放列表

> 源码：`lib/utils/m3u8_parser.dart`

### M3U8 格式简介

M3U8 是 HLS（HTTP Live Streaming）的播放列表格式。一个视频被切成很多小片段（.ts 文件），M3U8 文件记录了这些片段的地址。

#### Master 播放列表（多清晰度）

```
#EXTM3U
#EXT-X-STREAM-INF:BANDWIDTH=2000000,RESOLUTION=1280x720
https://cdn.example.com/720p/playlist.m3u8
#EXT-X-STREAM-INF:BANDWIDTH=4000000,RESOLUTION=1920x1080
https://cdn.example.com/1080p/playlist.m3u8
```

#### Media 播放列表（具体片段）

```
#EXTM3U
#EXT-X-VERSION:3
#EXT-X-TARGETDURATION:10
#EXTINF:10.0,
https://cdn.example.com/seg_00001.ts    ← 第1个片段（10秒）
#EXTINF:10.0,
https://cdn.example.com/seg_00002.ts    ← 第2个片段（10秒）
#EXTINF:10.0,
https://cdn.example.com/seg_00003.ts    ← 第3个片段（10秒）
#EXT-X-ENDLIST
```

### M3u8Parser 方法

| 方法 | 作用 |
|------|------|
| `detectType(content)` | 判断是 Master 还是 Media 播放列表 |
| `parseMasterPlaylist(content, baseUrl)` | 解析 Master 播放列表，提取所有清晰度变体 |
| `parseMediaPlaylist(content, baseUrl)` | 解析 Media 播放列表，提取所有 .ts 片段 |
| `resolveNestedSegments(segments, fetcher)` | 展开嵌套 M3U8（递归，最大深度 3） |
| `buildLocalM3u8(segments, ...)` | 构建本地 M3U8 文件（下载功能用） |
| `extractUniqueKeys(playlist)` | 提取加密密钥（有些视频是 AES 加密的） |

### 数据模型

```dart
// Master 播放列表中的一个清晰度变体
class M3u8Variant {
    int bandwidth;       // 带宽
    String? resolution;  // 分辨率，如 "1920x1080"
    String uri;          // 对应的 Media 播放列表 URL
}

// Media 播放列表中的一个片段
class M3u8Segment {
    double duration;           // 片段时长（秒）
    String uri;                // .ts 文件 URL
    int discontinuityGroup;    // 不连续组（广告过滤用）
    M3u8Key? key;              // 加密密钥（如果有）
}

// 加密信息
class M3u8Key {
    String method;  // 加密方法，如 "AES-128"
    String uri;     // 密钥 URL
    String? iv;     // 初始化向量
}
```

### 选择最高清晰度

```dart
M3u8Variant get bestVariant {
    return variants.reduce((a, b) => a.bandwidth > b.bandwidth ? a : b);
}
```

### 嵌套 M3U8 展开

有些网站的 M3U8 里套着 M3U8（片段的 URL 指向另一个 M3U8 文件），需要递归展开：

```
原始 M3U8：
  seg_001.ts
  seg_002.m3u8    ← 这不是 .ts，而是另一个 M3U8！
  seg_003.ts

展开后：
  seg_001.ts
  nested_001.ts   ← 从 seg_002.m3u8 中解析出来的
  nested_002.ts
  seg_003.ts
```

## 6.7 广告过滤

> 源码：`lib/utils/m3u8_ad_filter.dart`

M3U8 中的广告通常通过 `#EXT-X-DISCONTINUITY` 标记分隔。广告片段的特征：
- 时长异常（很短或很长）
- 在 `discontinuityGroup` 边界处
- 与正片的编码参数不同

`M3u8AdFilter` 通过启发式算法识别并移除这些广告片段。

## 6.8 第四步：播放器播放

> 源码：`lib/pages/player/player_controller.dart`

拿到视频 URL 后，构造播放参数，初始化播放器：

```dart
final params = PlaybackInitParams(
    videoUrl: source.url,           // 视频流 URL（M3U8 或 MP4）
    offset: source.offset,          // 续播位置（秒）
    isLocalPlayback: false,         // 是否本地播放
    bangumiId: bangumiItem.id,      // Bangumi ID（弹幕用）
    pluginName: currentPlugin.name, // 规则名
    episode: selection.episode,     // 集数
    httpHeaders: {                  // HTTP 头（防盗链）
        'user-agent': currentPlugin.userAgent,
        'referer': currentPlugin.referer,
    },
    adBlockerEnabled: true,         // 广告过滤
    episodeTitle: "...",            // 集标题
    referer: "...",                 // Referer
    currentRoad: selection.road,    // 当前播放源
    coverUrl: "...",                // 封面图
    bangumiName: "...",             // 番剧名
);

await playerController.init(params);
```

PlayerController 是一个组合控制器，内部管理 6 个子控制器：
- 播放控制（play/pause/seek）
- 弹幕控制（加载/显示/过滤）
- 手势控制（亮度/音量/进度）
- 屏幕控制（全屏/画中画）
- 历史记录（进度保存/恢复）
- 下载控制（缓存管理）

## 6.9 并发控制：_AsyncSessionOwner

> 源码：`lib/pages/video/video_controller.dart`

播放页有三类异步操作需要独立取消：

| Session Owner | 控制的操作 | 取消场景 |
|---------------|-----------|---------|
| `_playbackSessions` | 视频解析 + 播放器初始化 | 切换集数、退出播放页 |
| `_danmakuSessions` | 弹幕加载 | 切换集数、手动选择弹幕 |
| `_commentSessions` | 评论加载 | 切换集数、刷新评论 |

```dart
final _AsyncSessionOwner _playbackSessions = _AsyncSessionOwner();
final _AsyncSessionOwner _danmakuSessions = _AsyncSessionOwner();
final _AsyncSessionOwner _commentSessions = _AsyncSessionOwner();
```

每个异步操作开始时创建 session，完成后检查 session 是否还有效（没被取消）：

```dart
final session = _playbackSessions.begin();
// ... 异步操作 ...
if (session.isStale) return;  // 被取消了，丢弃结果
```

## 6.12 总结：从搜索到播放的完整链路

```
用户搜索 "进击的巨人"
    ↓
PluginSearchService.queryAllSource()     ← 并发调用所有规则
    ↓
Plugin.queryBangumi()                    ← HTML + XPath 提取搜索结果
    ↓
SourceSheet 显示搜索结果卡片              ← TabBar + Observer
    ↓ 用户点击
VideoPageController.queryRoads()         ← 获取剧集列表
    ↓
Plugin.querychapterRoads()               ← HTML + XPath 提取剧集链接
    ↓ 用户选集
VideoPageController.changeEpisode()      ← 触发视频提取
    ↓
WebViewVideoSourceService.resolve()      ← HeadlessWebView 拦截视频流
    ↓
M3u8Parser.parseMediaPlaylist()          ← 解析 .ts 片段列表
    ↓
PlayerController.init()                  ← mpv 播放器加载 M3U8
    ↓
视频播放 + 弹幕显示
```

### 核心设计思想

1. **规则与引擎分离**：JSON 描述结构，代码执行解析。新增网站只需写 JSON，不改代码。
2. **WebView 黑魔法**：用无头浏览器"骗"网站执行 JS，拿到动态生成的视频地址。
3. **平台抽象**：WebView 和播放器都有工厂模式，一套代码跑 Windows/Android/iOS/Linux。
4. **并发 + 取消**：Stream 推送搜索结果、Session 管理异步操作，切集/退出时优雅取消。
